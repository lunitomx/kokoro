   - `TestForcesSkillExists`

   - `TestForcesStructure`
     - `test_has_four_forces` — all 4 Customer Forces sections present (Trigger Event, Push, Inercia, Pull y Friccion)
     - `test_trigger_event_is_first_force` — "Trigger Event" appears as Fuerza 1
     - `test_inercia_present` — "Inercia" present as force/enemy
     - `test_forces_in_order` — forces appear in prescribed order (Fuerza 1 through Fuerza 4)
   - `TestForcesContent`
     - `test_has_eduardo_voice` — Eduardo vocabulary present (invitado, creacion, inversion — at least 2)
     - `test_references_knowledge_file` — references `kokoro-phase2-forces.md`
     - `test_has_anti_pattern_warnings` — anti-pattern language present (e.g., competing on features, generic desires)
     - `test_has_proyector_strategy` — Proyector strategy guidance (permiso or invitacion)

     - `test_each_force_has_guiding_questions` — each of 4 force sections contains at least 2 question marks
     - `test_has_customer_criteria` — "customer criteria" present in content
     - `test_has_momento_aja` — "momento aja" present in content
     - `test_has_oferta_mafia` — "oferta mafia" present in content
   - `TestForcesSummary`
     - `test_has_summary_template` — summary section with table template mapping all forces
   - `TestForcesCLI`
     - `test_init_copies_forces_skill` — `kokoro init` copies file to target `.claude/commands/`
     - `test_copied_skill_matches_source` — copied file matches source exactly

**Verify:**

- `uv run pytest tests/test_canvas.py` → PASS (no regressions)

### T2: GREEN — Create kokoro-forces.md skill file

**Goal:** Create the skill file with Eduardo's voice, all 4 Customer Forces in correct
order, guiding questions, Customer Criteria, Momento Aja, Oferta Mafia, anti-patterns,
and summary template. All T1 tests pass.

**Steps:**

1. Create `extension/.claude/commands/kokoro-forces.md` with:
   - Opening quote in Eduardo's voice: "El cliente rara vez compra lo que la empresa cree que esta vendiendo."
   - `## Contexto` — purpose, hero's journey (monomito) framing, invitado as hero, company as guide
   - `## Antes de Empezar — Estrategia del Proyector` — ask permission, set context, 70/30 rule
   - `## Fuerza 1: Trigger Event` — specific event definition, correct vs incorrect examples, guiding questions
   - `## Fuerza 2: Push` — achievement motivation, clear goal, guiding questions
   - `## Fuerza 3: Inercia` — main enemy definition, education strategy, guiding questions
   - `## Fuerza 4: Pull y Friccion` — pull elements (testimonios), friction elements (anxieties), strategy, guiding questions
   - `## Customer Criteria` — distinctive experience characteristics, not features
   - `## Momento Aja` — optimal receptivity window, timing of messages
   - `## La Oferta Mafia` — 5-step irresistible offer structure using all 4 forces
   - `## Resumen de Fuerzas` — markdown table template mapping all forces + criteria + offer
   - Reference to `kokoro-phase2-forces.md` knowledge file
   - Anti-pattern warnings (generic desires vs concrete triggers, competing on features vs addressing inertia)
   - Eduardo's vocabulary throughout (invitado, creacion, inversion — never cliente, producto, precio)
2. Each force section includes:
   - Explanation of what the force captures
   - At least 2 guiding questions (powerful questions, not prescriptions)
   - Eduardo's coaching perspective
   - Interactive prompt: wait for entrepreneur's answer before proceeding

**Verify:**
- `uv run pytest tests/test_forces.py` → PASS (all GREEN)
- `uv run pytest` → PASS (full suite, no regressions)
- `uv run pyright` → PASS
- `uv run ruff check` → PASS

**Commit:** `feat(s2.3): T2 — GREEN kokoro-forces.md skill file with Eduardo's voice`
