# S2.2 Design — /kokoro-canvas Skill

## Problem & Value

**Problem:** Entrepreneurs need a structured way to map their business model before building. Without guidance, they start with the solution (Eduardo's #1 anti-pattern) instead of the segment/problem. This leads to biased models where the segment and problem are reverse-engineered to justify a pre-existing solution — the "inciensos de ballenas" trap.

**Value:** Eduardo guides the entrepreneur through a Lean Canvas segment-first, ensuring they validate the right things in the right order. Output is a completed canvas they can reference throughout Phase 2. This is the first Phase 2 skill — the foundation for forces analysis, interviews, and validation planning.

## Approach

Create `extension/.claude/commands/kokoro-canvas.md` — a markdown prompt file that Claude Code executes when a user types `/kokoro-canvas`. The prompt guides the user interactively through 9 Lean Canvas blocks in Eduardo's recommended order (segment-first, solution LAST).

The skill follows Eduardo's coaching methodology: ask before guiding (Proyector strategy), listen 70%/talk 30%, use powerful questions rather than prescriptions.

The skill file references `kokoro-phase2-canvas.md` as knowledge context. No CLI changes needed — `_copy_dir_no_overwrite` already handles `commands/`.

## Fill Order

Eduardo's methodology enforces a specific fill order — never starting with the solution:

| Order | Block | Rationale |
|-------|-------|-----------|
| 1 | Segmento de Invitados | Who you serve — including early adopters |
| 2 | Problema | The 1-3 real problems of the segment |
| 3 | Propuesta Unica de Valor | The center of the canvas — measurable promise |
| 4 | Ventaja Injusta | What cannot be easily copied or bought |
| 5 | Canales | All means of delivering value to invitados |
| 6 | Flujo de Ingresos | What you sell + investment + delivery model |
| 7 | Estructura de Costos | All expense concepts (not amounts) |
| 8 | Metricas Clave | 2-3 numbers that show business health |
| 9 | Solucion | LAST — possible solution for each identified problem |

Anti-pattern: Starting with Block 9 (Solution) biases the entire model. Eduardo's "inciensos de ballenas" example demonstrates this trap.

## Acceptance Criteria

**MUST:**
1. `kokoro-canvas.md` exists in `extension/.claude/commands/`
2. Contains all 9 Lean Canvas blocks with guiding questions
3. Enforces segment-first order (solution is last block)
4. Eduardo's voice present (vocabulary table words, Proyector strategy, anti-patterns)
5. References knowledge file `kokoro-phase2-canvas.md`
6. Includes summary template covering all 9 blocks
7. `kokoro init` copies the skill file to target `.claude/commands/`

**SHOULD:**
8. Each block has at least 2 guiding questions
9. Anti-pattern warnings for common mistakes per block

**MUST NOT:**
10. Must not start with solution block
11. Must not use generic marketing language (hacks, growth, monetizar)

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Fill order: Segmento > Problema > PUV > Ventaja Injusta > Canales > Ingresos > Costos > Metricas > Solucion | Eduardo explicitly warns against solution-first. Source material shows segment-first as critical correction. | Standard L-R order, Ash Maurya original order |
| D2 | Interactive guidance (block by block, wait for answers) | Eduardo's coaching: 70% listening, 30% talking. Proyector strategy. | Full canvas dump with fill-in-the-blank |
| D3 | Summary template in markdown table format | Consistent with kokoro-finance.md summary pattern. Easy copy-paste. | Free-form text, YAML output |
| D4 | Single markdown file following PAT-L-001 | Proven pattern from S1.4-S1.7. No CLI changes needed. | Multi-file skill |
