# S2.1 Knowledge Files Phase 2

> **Epic:** E2 — Kokoro Phase 2: Elegir la Semilla
> **Size:** S
> **Status:** In Progress

## User Story (Connextra)

**As** an entrepreneur using Kokoro,
**I want** Phase 2 methodology references (Lean Canvas, Customer Forces, validation interviews, experiment planning) embedded in the extension,
**So that** when I use Phase 2 skills, the AI guides me with Eduardo's full methodology — not generic business advice.

## Acceptance Criteria (Gherkin)

### AC1: Knowledge files exist and follow template structure

```gherkin
Given the kokoro extension is initialized
When I inspect the .claude/knowledge/ directory
Then I find 4 Phase 2 knowledge files:
  | File                              | Covers                          |
  | kokoro-phase2-canvas.md           | Lean Canvas methodology         |
  | kokoro-phase2-forces.md           | Customer Forces Model           |
  | kokoro-phase2-interviews.md       | Validation interviews           |
  | kokoro-phase2-validation.md       | Validation plan & experiments   |
And each file follows PAT-L-008 template structure:
  - kokoro-managed header comment
  - Title with skill reference
  - Proposito section
  - Methodology sections with actionable guidance
  - Conexion con Otros Skills section
```

### AC2: Knowledge files installed by kokoro init

```gherkin
Given the kokoro extension package is installed
When I run kokoro init
Then all 4 Phase 2 knowledge files are copied to .claude/knowledge/
And they have the kokoro-managed header
```

### AC3: Content reflects Eduardo's methodology

```gherkin
Given a Phase 2 knowledge file
When I review its content
Then it contains methodology from the source material (02_Modulo_Semilla)
And it uses Eduardo's vocabulary (inversión, creación, invitado)
And it avoids anti-pattern language (precio, producto, cliente, gratis)
```

### AC4: Shared test fixtures via conftest.py

```gherkin
Given the test suite for kokoro
When tests need common fixtures (paths, file readers, content validators)
Then they import from conftest.py
And fixture duplication across test modules is eliminated
```
