# S2.2: /kokoro-canvas — User Story

> **Status:** In Progress
> **Size:** M
> **Created:** 2026-03-10

## User Story (Connextra)

**As** an entrepreneur using Kokoro Phase 2,
**I want** a guided Lean Canvas session led by Eduardo,
**So that** I can validate my business model starting from customer segments — never from the solution.

## Acceptance Criteria (Gherkin)

### AC1: Skill file exists at correct path

```gherkin
Given the kokoro extension is initialized
When I check for the canvas skill file
Then `extension/.claude/commands/kokoro-canvas.md` exists
```

### AC2: Contains all 9 Lean Canvas blocks

```gherkin
Given the skill file exists
When I inspect its content
Then it contains all 9 Lean Canvas blocks:
  | Block                    |
  | Segmento                 |
  | Problema                 |
  | Propuesta Unica de Valor |
  | Solucion                 |
  | Canales                  |
  | Flujo de Ingresos        |
  | Estructura de Costos     |
  | Metricas Clave           |
  | Ventaja Injusta          |
```

### AC3: Enforces segment-first fill order

```gherkin
Given the skill file exists
When I inspect its content
Then it enforces segment-first fill order
And it explicitly states to never start with solution
And the fill sequence begins with Segmento, then Problema
```

### AC4: Uses Eduardo's voice

```gherkin
Given the skill file exists
When I inspect its content
Then it uses Eduardo's vocabulary (e.g., "semilla", "raiz", "proyector")
And it includes anti-patterns to avoid
And it follows Proyector strategy guidance
```

### AC5: References knowledge file

```gherkin
Given the skill file exists
When I inspect its content
Then it references `kokoro-phase2-canvas.md` knowledge file
```

### AC6: Includes summary template

```gherkin
Given the skill file exists
When I inspect its content
Then it includes a summary template for entrepreneur output
And the template covers all 9 Lean Canvas blocks
```

### AC7: kokoro init copies the file correctly

```gherkin
Given the kokoro-canvas.md skill file exists in extension/
When `kokoro init` is executed
Then the file is copied to the target project's .claude/commands/
And the file content is preserved exactly
```
