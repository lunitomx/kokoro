# S2.3 Design — /kokoro-forces Skill

## Problem & Value

**Problem:** Entrepreneurs who completed their Lean Canvas have a model, but they do not understand the hidden psychological forces driving their customers' buying decisions. Without mapping these forces, they compete on features and price instead of addressing the real reasons people switch (or don't). Eduardo identifies inertia — not competition — as the true enemy.

**Value:** A guided Customer Forces analysis session maps the 4 forces (Trigger Event, Push, Inercia, Pull/Friccion) so the entrepreneur understands WHY their invitados buy. This feeds directly into interviews (S2.4) and validation planning. The Oferta Mafia structure synthesizes the forces into an irresistible offer narrative.

## Approach

Create `extension/.claude/commands/kokoro-forces.md` — a markdown prompt file executed when a user types `/kokoro-forces`. The prompt guides the user through the 4 Customer Forces, Customer Criteria, Momento Aja, and Oferta Mafia construction, all in Eduardo's voice.

Follows the proven pattern from S2.2 (PAT-L-001 + PAT-L-006): single markdown skill file with content validated by structural tests. No CLI changes needed — `_copy_dir_no_overwrite` already handles `commands/`.

The skill references `kokoro-phase2-forces.md` as knowledge context. Eduardo's coaching methodology applies: Proyector strategy (ask before guiding), listen 70%/talk 30%, powerful questions per section.

## Content Structure

The skill file organizes around the 4 forces plus synthesis sections:

| Section | Content | Key Concepts |
|---------|---------|-------------|
| Contexto | Purpose, hero's journey framing | Monomito, invitado as hero, company as guide |
| Estrategia del Proyector | Ask permission before guiding | Proyector strategy, invitation |
| Trigger Event | Specific event that triggers search | Concrete moment, not general desire |
| Push | Achievement motivation driving action | Clear goal, leaving comfort zone |
| Inercia | Habits + anxieties resisting change | True enemy, education as strategy |
| Pull/Friccion | Attraction elements + resistance | Testimonios, objection handling |
| Customer Criteria | Distinctive experience details | Not features — experience differentiators |
| Momento Aja | Optimal receptivity window | Timing of messages |
| Oferta Mafia | Irresistible offer structure | 5-step narrative using all 4 forces |
| Resumen | Forces map template | All forces + criteria + offer in one view |

## Acceptance Criteria

**MUST:**
1. `kokoro-forces.md` exists in `extension/.claude/commands/`
2. Contains all 4 Customer Forces sections with guiding questions
3. Enforces correct force order (Trigger Event, Push, Inercia, Pull/Friccion)
4. Eduardo's voice present (vocabulary words, Proyector strategy, anti-patterns)
5. References knowledge file `kokoro-phase2-forces.md`
6. Includes Customer Criteria and Momento Aja sections
7. Includes Oferta Mafia structure (5-step narrative)
8. Includes summary template mapping all forces
9. `kokoro init` copies the skill file to target `.claude/commands/`

**SHOULD:**
10. Each force section has at least 2 guiding questions
11. Anti-pattern warnings (e.g., generic desires vs concrete triggers, competing on features vs addressing inertia)
12. Monomito / hero's journey framing in context section

**MUST NOT:**
13. Must not use generic marketing language (hacks, growth, monetizar)
14. Must not position company as hero (invitado is always the hero)
15. Must not use "cliente" instead of "invitado", "precio" instead of "inversion", "producto" instead of "creacion"

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | 4 force sections, not combined | Knowledge file treats each force distinctly with its own examples and strategies. Combining would lose specificity. | Combined Push+Pull section, single "forces overview" |
| D2 | Pull and Friccion in one section (Fuerza 4) | Knowledge file pairs them as "attraction vs resistance" — opposite sides of the same decision moment. Scope doc lists them together. | Separate sections for Pull and Friccion |
| D3 | Oferta Mafia as synthesis section at end | The Oferta Mafia explicitly uses all 4 forces in its 5-step structure. Must come after forces are mapped. | Oferta Mafia as separate skill |
| D4 | Single markdown file following PAT-L-001 | Proven pattern from S2.2. No CLI changes needed. | Multi-file skill |
| D5 | Test file mirrors test_canvas.py structure | PAT-L-006: consistent test patterns reduce cognitive load and enable copy-adapt workflow. | New test structure |
