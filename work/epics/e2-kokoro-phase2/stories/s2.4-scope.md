# S2.4 Scope — /kokoro-interviews

> **Story:** S2.4 | **Epic:** E2 | **Size:** M

## User Story

As an entrepreneur with a Lean Canvas and Customer Forces mapped,
I want a guided validation interview process
So that I can validate my assumptions with real data from potential customers.

## In Scope

- Skill file `extension/.claude/commands/kokoro-interviews.md`
- Problem Interview section (preparation, structure, questions)
- Solution Interview section (when, structure, validation)
- Post-Interview Analysis section (patterns, persona update, forces canvas)
- Coaching methodology (coach not consultant)
- Bias awareness section
- Quality metrics for interview rounds
- Eduardo's voice throughout
- Guiding questions per section
- Summary template
- Reference to `kokoro-phase2-interviews.md` knowledge file
- TDD with structural content validation
- CLI integration: `kokoro init` copies file

## Out of Scope

- Actual interview execution tools
- Recording/transcription features
- CRM integration
- Automated pattern analysis

## Done When

- Skill file exists at correct path
- Problem and Solution interview sections present
- Analysis methodology present
- Eduardo's voice markers present (>= 3 terms)
- Anti-pattern warnings present
- Summary template with >= 4 table rows
- `kokoro init` copies file correctly
- All tests pass (new + existing)
- pyright + ruff clean
