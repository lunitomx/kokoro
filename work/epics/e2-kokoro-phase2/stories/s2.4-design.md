# S2.4 Design — /kokoro-interviews Skill

## Complexity Assessment

**Size: M** — More content sections than forces (Problem Interview, Solution Interview,
Post-Interview Analysis, Coaching Methodology, Bias Awareness, Quality Metrics, Summary).
The knowledge file is the longest of the Phase 2 files with deeper coaching methodology.
3 tasks expected (RED, GREEN, REFACTOR).

## Problem & Value

**Problem:** Entrepreneurs who completed their Lean Canvas and Customer Forces have a model
and understand the psychological forces, but they have never validated these assumptions with
real people. Without structured interviews, they build on speculation. Eduardo emphasizes:
"No midas lo que los consumidores dicen que haran, mide lo que hicieron." Most businesses
die in the gap between intention and action.

**Value:** A guided interview process teaches the entrepreneur to conduct Problem Interviews
(discover real problems) and Solution Interviews (validate with prototypes), with bias
awareness and post-interview analysis that feeds back into the Canvas and Forces. This is
the validation engine that turns assumptions into evidence.

## Approach

Create `extension/.claude/commands/kokoro-interviews.md` — a markdown prompt file executed
when a user types `/kokoro-interviews`. The prompt guides the user through Problem Interview
preparation and execution, Solution Interview timing and structure, Post-Interview Analysis,
coaching methodology, bias awareness, and quality metrics, all in Eduardo's voice.

Follows the proven pattern from S2.2 and S2.3 (PAT-L-001 + PAT-L-006): single markdown
skill file with content validated by structural tests. No CLI changes needed —
`_copy_dir_no_overwrite` already handles `commands/`.

The skill references `kokoro-phase2-interviews.md` as knowledge context. Eduardo's coaching
methodology applies: Proyector strategy (ask before guiding), coach not consultant, listen
70%/talk 30%, powerful questions per section.

## Content Structure

The skill file organizes around interview phases plus coaching and analysis sections:

| Section | Content | Key Concepts |
|---------|---------|-------------|
| Contexto | Purpose, behaviors over intentions | Validation as evidence, not confirmation |
| Estrategia del Proyector | Ask permission before guiding | Proyector strategy, invitation |
| Entrevista de Problema — Preparacion | Selecting invitados, invitation message, 20-min ask | No selling, expertise framing |
| Entrevista de Problema — Estructura | 4-step session (context, narrative, active listening, close) | 70/30 rule, silence as tool |
| Preguntas Poderosas | Questions for problem discovery + Customer Forces | Past behavior focus, concrete events |
| Lo que NUNCA hacer | Anti-patterns during interviews | No selling, no leading questions, no future intentions |
| Entrevista de Solucion | When to do it (after 10 problem interviews), structure, observation | Behaviors > words, prototype validation |
| Analisis Post-Entrevistas | Pattern identification, persona update, forces canvas update | 10-interview threshold, real vocabulary |
| Sesgos a Vigilar | Confirmation, social desirability, recency, availability biases | Self-awareness during interviews |
| Metodologia de Coaching | Coach not consultant, question-first approach | "Has considerado?" not "Yo lo que haria..." |
| Metricas de Calidad | What a good interview round produces | Patterns, real vocab, unexpected insights |
| Resumen | Interview preparation checklist + findings template | All sections in one structured view |

## Acceptance Criteria

**MUST:**
1. `kokoro-interviews.md` exists in `extension/.claude/commands/`
2. Contains Problem Interview section with preparation, structure, and powerful questions
3. Contains Solution Interview section with timing prerequisite and structure
4. Contains Post-Interview Analysis section (patterns, persona update, forces canvas update)
5. Eduardo's voice present (>= 3 vocabulary terms: invitado, creacion, inversion)
6. References knowledge file `kokoro-phase2-interviews.md`
7. Contains anti-pattern warnings (no selling, no leading questions, no future intentions)
8. Contains Proyector strategy section (permiso or invitacion)
9. Contains coaching methodology section (coach not consultant)
10. Contains bias awareness section (confirmation, social desirability, recency, availability)
11. Contains quality metrics section
12. Includes summary template with >= 4 table rows
13. `kokoro init` copies the skill file to target `.claude/commands/`
14. Each major section has at least 2 guiding questions

**SHOULD:**
15. Guiding questions focus on past behavior, not future intentions
16. Connection to other skills (canvas, forces, validate)
17. 70/30 listening rule emphasized

**MUST NOT:**
18. Must not use generic marketing language (hacks, growth, monetizar)
19. Must not position company as hero (invitado is always the hero)
20. Must not use "cliente" instead of "invitado", "precio" instead of "inversion", "producto" instead of "creacion"

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Problem Interview and Solution Interview as separate major sections | Knowledge file treats them as distinct phases with different prerequisites (solution requires 10 problem interviews first). Combining would blur the sequencing. | Single "Interview" section, separate skill per interview type |
| D2 | Bias awareness as dedicated section, not inline | 4 distinct biases deserve explicit treatment. Inline mentions would be lost in longer sections (U-shaped attention, BASE-055). | Inline bias warnings per section |
| D3 | Coaching methodology as standalone section | The coach-not-consultant principle is a meta-skill that applies to ALL interview interactions, not just one phase. Deserves prominence. | Inline coaching notes per section |
| D4 | Single markdown file following PAT-L-001 | Proven pattern from S2.2 and S2.3. No CLI changes needed. | Multi-file skill |
| D5 | Test file mirrors test_forces.py structure with expanded sections | PAT-L-006: consistent test patterns reduce cognitive load. M size justifies ~15-18 tests (more sections than forces). | Minimal test file, separate test files per section |
| D6 | Summary template with >= 4 rows (QR learning from S2.3) | PAT-L-012: weak assertions pass tests but fail semantic intent. 4 rows ensures meaningful coverage of interview phases. | 2-row minimum |
