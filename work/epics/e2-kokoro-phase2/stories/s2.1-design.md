# S2.1 Design — Knowledge Files Phase 2

> **Story:** S2.1 | **Epic:** E2 | **Complexity:** Simple

## Problem

Phase 2 skills (S2.2-S2.5) need methodology reference files so the AI guides
entrepreneurs through Lean Canvas, Customer Forces, interviews, and validation
using Eduardo's full methodology — not generic business advice. Without
knowledge files, skills would need to embed all methodology content inline,
violating separation of concerns.

## Value

Skills can reference methodology details from knowledge files, keeping skill
files focused on guided interaction. Entrepreneurs get accurate Eduardo
methodology with his vocabulary and anti-patterns enforced.

## Approach

Create 4 knowledge files following PAT-L-008 structure (established by E1's
kokoro-phase1-*.md files). Also create `tests/conftest.py` with shared fixtures
to address E1 retrospective finding about fixture duplication.

### Knowledge Files

All files follow the same template:

```markdown
<!-- kokoro-managed: do not edit, will be overwritten by kokoro init -->
# {Title}

> Skill: `/kokoro-{skill}`
> Fase 2: Elegir la Semilla

## Proposito
...

## {Methodology Sections}
...

## Conexion con Otros Skills
...
```

#### 1. kokoro-phase2-canvas.md — Lean Canvas

Source: 02_Modulo_Semilla sections on Lean Canvas Model (lines 432-668).

Key content:
- 3 questions every model must answer (create value, deliver value, capture value)
- 9 blocks with correct fill order: segment first, NEVER start with solution
- Rules: fits on one A4, max 1 hour, fill physically
- Propuesta Unica de Valor formula: resultado + periodo + eliminacion de objeciones
- Ventaja Injusta criteria
- Risk analysis: producto, consumidor, mercado
- Eduardo vocabulary: inversion not precio, creacion not producto

#### 2. kokoro-phase2-forces.md — Customer Forces Model

Source: 02_Modulo_Semilla sections on Customer Forces (lines 670-1030).

Key content:
- 4 forces: Trigger Event, Push, Inertia, Pull/Friction
- Trigger = specific event that violates expectations, not a general desire
- Push = clear achievement motivation to leave comfort zone
- Inertia = the real enemy (not competition), combated with education
- Pull = external attraction (testimonials, before/after, guarantees)
- Friction = fears, anxieties, uncertainties that stop purchase
- Connection to Mafia Offer structure (recognize, identify, name, differentiate, handle)
- Customer criteria and the "Momento Aja"
- Best selling moments: status quo > consideration > post-purchase

#### 3. kokoro-phase2-interviews.md — Validation Interviews

Source: 02_Modulo_Semilla interview methodology sections (lines 547-582, 978-993).

Key content:
- Problem interview structure: preparation, invitation, context, listen 70%, close
- Message: "No es para venderte, necesito tu expertise para feedback"
- 15 minutes active listening, take extensive notes, sell nothing
- Post-interview analysis: patterns across 10 interviews
- Solution interview: present findings, validate with prototype/concept
- Customer Forces discovery questions per force
- Rule: measure what consumers DID, not what they SAY they will do
- Behaviors > intentions

#### 4. kokoro-phase2-validation.md — Validation Plan & Experiments

Source: 02_Modulo_Semilla 3x3x3 model and risk sections (lines 250-266, 596-624).

Key content:
- 3x3x3 model: 3 years vision, 3 months strategy, 3 weeks sprints
- Build-Measure-Learn cycle per sprint
- Risk identification from Lean Canvas (product, consumer, market risks)
- Attack riskiest assumption first
- Evidence-based decisions (principle 7)
- Qualitative validates, quantitative verifies (principle 8)
- Experiment design: hypothesis, metric, threshold, duration
- Connection to Montaña del Mañana (Phase 1) for 3-year anchor

## Acceptance Criteria

### MUST

|-----|-----------|
| AC1 | 4 knowledge files exist at `extension/.claude/knowledge/kokoro-phase2-{canvas,forces,interviews,validation}.md` |

| ID  | Criterion |
|-----|-----------|

### MUST NOT

| ID  | Criterion |
|-----|-----------|

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Follow PAT-L-008 identical to E1 knowledge files | Proven pattern, consistency across phases |
| D2 | One knowledge file per skill (1:1 mapping) | Each S2.2-S2.5 skill loads its own knowledge reference |
| D3 | conftest.py with path fixtures only | Simple First — address clear duplication, add helpers later if needed |
| D4 | Canvas: segment-first fill order | Eduardo's core anti-pattern: never start with solution |
| D5 | Forces: include Mafia Offer connection | Source material treats forces and offer as connected concepts |
