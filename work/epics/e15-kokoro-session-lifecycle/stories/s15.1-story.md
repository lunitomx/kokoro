# S15.1 — Session log model

## Objective

Define the data structure for per-client session logs that /kokoro-open
and /kokoro-close will read and write. Document the schema in a knowledge
file so all Kokoro skills follow the same format.

## Acceptance Criteria

- [ ] Session log schema documented in knowledge file
- [ ] Schema supports: date, type, summary, hallazgos, artifacts, next_action
- [ ] No changes to ClientProfile Pydantic model (uses metadata)
- [ ] Knowledge file has extension mirror
