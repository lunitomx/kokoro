# S32.1 — Knowledge: Lean Landing Page

## User Story

As Eduardo (strategy consultant), I want the Lean Landing Page methodology
documented as a structured knowledge file, so that `/kokoro-landing` (and other
skills like `/kokoro-launch`) can reference it when evaluating or generating
landing pages.

## Acceptance Criteria

1. **Knowledge file covers 5 principles with verifiable criteria**
   - Each principle has a clear definition, what to look for, and how to score
     (present / partial / absent)

2. **Knowledge file covers 9 blocks with golden rules per block**
   - Each block documents: objective, golden rule, what to evaluate, anti-pattern,
     and example reference

3. **Anti-patterns documented**
   - What a Lean Landing Page is NOT (brochure, blog, institutional page)
   - Common mistakes per block

4. **Reference metrics included**
   - Conversion to appointment: 25%
   - ICP Match quality: 90% High Fit
   - Show-up rate: 80%
   - Time to decision: 7 days

5. **Installed in both mirrors**
   - `extension/.claude/knowledge/kokoro-lean-landing.md`
   - `.claude/knowledge/kokoro-lean-landing.md`

6. **Format consistent with existing knowledge files**
   - Use `kokoro-meta-ai-ecosystem.md` as structural reference
   - YAML frontmatter, clear sections, actionable content
