### Task 2 (M): Write the 5-step evaluation process

**Goal:** Write the core audit methodology as 5 sequential pasos.

**Steps:**
1. **Paso 1 — Obtener la landing:** 3 input modes:
   - URL via WebFetch
   - Pasted HTML
   - Consultive mode (user describes the landing verbally)
2. **Paso 2 — Mapeo de bloques:** Table of 9 blocks with definitions:
   - 01 HERO, 02 PAINS, 03 INERCIAS, 04 ASI FUNCIONA,
     05 MAFIA OFFER, 06 NO ES PARA TI SI, 07 FRICTIONS,
     08 QUID PRO QUO, 09 CTA PERSISTENTE
   - Output: block map table (presente/ausente/parcial)
3. **Paso 3 — Evaluacion por bloque:** For each present/partial block:
   - Score: fuerte / parcial / debil
   - Copy actual (quote from the landing)
   - Rewrite sugerido (Eduardo voice, vocabulario luxurizante)
   - Diagnostico (what is missing, why it matters)
4. **Paso 4 — Scorecard de principios:** 5 principles from methodology:
   - Claridad en 10-20 segundos
   - Riesgo invertido
   - Filtro de prospectos
   - Proceso visible
   - CTA unico y persistente
   - Scoring: cumple / parcial / no cumple + evidencia
5. **Paso 5 — Diagnostico integrado:**
   - Missing blocks summary
   - Disorder in decision sequence
   - Top 3 acciones inmediatas
   - Suggestion to run `/kokoro-audit` for technical complement

**Files:** `extension/.claude/commands/kokoro-landing.md` (append pasos section)

**Verify:**
- All 5 pasos present with clear instructions
- All 9 blocks listed in Paso 2
- All 5 principles listed in Paso 4
- Per-block diagnosis format defined in Paso 3
- `/kokoro-audit` suggestion in Paso 5

**Commit:** Yes — `feat(s32.2): write 5-step evaluation process`

### Task 3 (S): Write output structure + anti-patrones + persistencia

**Goal:** Complete the command file with structured output template, anti-patrones,
and persistencia sections.

**Steps:**
1. Write `## Estructura de salida` with output template examples:
   - Block map table (matching design spec format)
   - Principle scorecard table
   - Per-block diagnosis format
   - Top 3 acciones inmediatas
2. Write `## Anti-patrones` section:
   - No generic feedback ("se ve bien")
   - No promising exact conversion results
   - No ignoring invitado context
   - No using forbidden vocabulary (producto, precio, cliente)
   - No giving tip lists instead of guided process
   - No evaluating visual/UX aspects (that is `/kokoro-audit`)
3. Write `### Persistencia` section:
   - Path: `invitados/{grupo}/landing/audit-{fecha}-{descripcion}.md`

**Files:** `extension/.claude/commands/kokoro-landing.md` (append sections)

**Verify:**
- Output template includes all 4 sections from design (block map, scorecard, per-block, top 3)
- Anti-patrones section present with minimum 5 anti-patterns
