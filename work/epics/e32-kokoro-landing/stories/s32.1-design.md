# S32.1 Design — Lean Landing Page Knowledge File

## Problem & Value

Eduardo's Lean Landing Page methodology exists only in a PDF presentation and a
class transcript. No Kokoro skill can reference it programmatically. Without a
structured knowledge file, `/kokoro-launch` generates landing pages without the
9-block decision flow, and the future `/kokoro-landing` (S32.2) has no
foundation to evaluate or score pages against.

**Value:** A single knowledge file makes the methodology available to any skill
that touches landing pages — evaluation, generation, scoring, auditing.

## Approach

Single knowledge file extraction. PDF is the primary source (structured,
authoritative). Transcript is secondary (adds nuance, real-world examples,
Eduardo's voice). Output matches the format of existing knowledge files
(see `kokoro-meta-ai-ecosystem.md`).

- Extract from PDF: 5 principles, 9 blocks, anti-patterns, metrics, use cases
- Enrich from transcript: examples, phrasings, clarifications Eduardo gave in class
- Write to `extension/.claude/knowledge/kokoro-lean-landing.md`
- Mirror to `.claude/knowledge/kokoro-lean-landing.md` (identical copy)

## Components

| Action | Path | Description |
|--------|------|-------------|
| create | `extension/.claude/knowledge/kokoro-lean-landing.md` | Primary knowledge file |
| create | `.claude/knowledge/kokoro-lean-landing.md` | Mirror copy (identical) |

No code changes. No skill modifications. No test changes.

## Acceptance Criteria

| ID | Criterion | Verifiable |
|----|-----------|------------|
| AC1 | Knowledge file contains all 5 principles with definition, criteria, and scoring per principle | Yes |
| AC2 | Knowledge file contains all 9 blocks with objective, golden rule, evaluation criteria, anti-pattern, and example per block | Yes |
| AC3 | Anti-patterns section documents what a Lean Landing Page is NOT plus common mistakes | Yes |
| AC4 | Reference metrics table includes all 4 benchmarks (conversion 25%, ICP 90%, show-up 80%, time 7d) | Yes |

## Decisions

### D1 — PDF as primary source, transcript as secondary

**Choice:** Use the PDF (14 slides, structured) as the authoritative source for
principles, blocks, and metrics. Use the transcript as enrichment for examples,
nuance, and Eduardo's teaching voice.

**Rationale:** The PDF is a curated v2.0 document — already structured, reviewed,
and organized by Eduardo. The transcript is a live class recording — rich in
context but unstructured. Extracting structure from the PDF and color from the
transcript produces the highest-quality knowledge file with the least risk of
misinterpretation.

**Alternatives considered:**
- Transcript-first (too unstructured, risk of missing blocks)
- PDF-only (loses real-world examples and teaching nuance)
- Merge both equally (creates conflict when they diverge)
