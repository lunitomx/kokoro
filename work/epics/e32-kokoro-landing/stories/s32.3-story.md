# S32.3 — Tests + verificacion

## User Story

As Eduardo (Kokoro operator), I want verification that `/kokoro-landing`
produces correct, vocabulary-compliant output, so that the skill is reliable
for production use with real entrepreneurs.

## Acceptance Criteria

```gherkin
Feature: /kokoro-landing verification

  Scenario: Vocabulary compliance
    Given the kokoro-landing.md command file
    When I check vocabulary
    Then no forbidden words appear in instructional content

  Scenario: Output structure completeness
    Given the structured output template
    When I verify completeness
    Then it contains block map + scorecard + per-block diagnosis + top 3 actions

  Scenario: Knowledge file reference accuracy
    Given kokoro-landing.md and kokoro-lean-landing.md
    When I compare block/principle references
    Then all 9 blocks and 5 principles are accurately represented

  Scenario: Integration suggestions
    Given the command file
    When I check integration suggestions
    Then /kokoro-audit is suggested as complement (not dependency)
```
