# Design: S13.1 — /kokoro-audit

## Decision: Skill Pattern
Follow the established Kokoro skill pattern from `kokoro-ads.md`:
- Header with credit line
- Client resolution section (from `.kokoro/clients.json`)
- Proyector strategy gate
- Sequential exercises (identify site, run audit, translate, prioritize)
- Output template with summary table + next steps
- Persistencia section

## Decision: Web-Quality Skills Integration
The skill references existing web-quality-skills as MCP tools (if available)
or guides manual execution. It does NOT re-implement any audit logic.

Skills wrapped:
- `seo-technical` / `seo-page` — SEO checks
- `performance` / `core-web-vitals` — Speed and rendering
- `accessibility` — WCAG compliance
- `best-practices` — Security and code quality

## Decision: Translation Table
Each technical finding maps to a business-impact explanation.
The skill includes a reference table (LCP, CLS, meta tags, HTTPS, etc.)
with plain-language translations for non-technical entrepreneurs.

## Decision: Priority Levels
Three tiers: Urgente (affects revenue now), Importante (SEO/UX), Deseable (refinements).
