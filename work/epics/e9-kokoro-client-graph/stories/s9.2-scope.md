# S9.2 Scope — Client Persistence Layer

## In Scope
- `load_registry(project_dir)` → `ClientRegistry | None`
- `save_registry(project_dir, registry)` → atomic write to `.kokoro/clients.json`
- `create_empty_registry()` → fresh `ClientRegistry` with timestamps
- Tests covering round-trip, missing file, overwrite, empty creation

## Out of Scope
- CLI commands for client management (future story)
- Migration from other formats
- Client search/query beyond what ClientRegistry already provides
- Multi-file storage or database backends
