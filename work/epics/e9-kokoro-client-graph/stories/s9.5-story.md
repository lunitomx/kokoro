# Story: Sync client graph with Rai memory

## User Story
As Kokoro,
I want to generate reference_*.md memory files from the client registry,
so that Rai's memory system stays in sync with the client graph.

## Acceptance Criteria

### Scenario: Generate reference file from ClientProfile
```gherkin
Given a ClientProfile for "Konecta Park" with repos, segments, metadata
When sync_to_memory is called
Then a reference_konecta_park.md file is generated with proper frontmatter
```

### Scenario: Don't overwrite manual edits
```gherkin
Given a reference file already exists
When sync_to_memory is called without force flag
Then the existing file is NOT overwritten
```

### Scenario: Update MEMORY.md index
```gherkin
Given a new reference file was generated
When sync_to_memory completes
Then MEMORY.md has a line pointing to the new file
```

## Notes
- Output dir: ~/.claude/projects/-Users-soyahuehuetedigital-Documents-RaizAncestral/memory/
- Follow existing reference file format (frontmatter with name, description, type: reference)
- Function: sync_to_memory(registry, memory_dir, force=False)
- Only generates for clients that don't have a reference file yet (unless force=True)
