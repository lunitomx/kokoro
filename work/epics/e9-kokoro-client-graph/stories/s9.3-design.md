# S9.3 Design — Skill /kokoro-client

## Decisions

### D1: Command file only, no knowledge file
This skill is a CRUD wrapper over models.py and store.py. No methodology
to document — unlike kokoro-ads which needed meta specs.

### D2: Conversational flow
Eduardo guides client creation through conversation, not a form.
Ask one question at a time, reflect back, confirm before saving.

### D3: Four operations
- **Crear** — Gather client info conversationally, save to registry
- **Listar** — Show all clients grouped by group
- **Ver** — Show full profile for one client (fuzzy match by name)
- **Buscar** — Search by segment, industry, or name

### D4: Uses store.py directly
- `load_registry()` / `save_registry()` / `create_empty_registry()`
- `ClientProfile` / `ClientRegistry` from models.py
- No new Python code needed — the skill is a prompt
