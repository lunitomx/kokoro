# S9.2 Design — Client Persistence Layer

## Decisions

### D1: Follow ontology/store.py pattern
Mirror the exact structure from `src/kokoro/ontology/store.py`:
- `load_registry(project_dir)` returns `ClientRegistry | None`
- `save_registry(project_dir, registry)` writes atomically
- `create_empty_registry()` returns fresh registry with timestamps

### D2: File location
Path: `{project_dir}/.kokoro/clients.json`
Same `.kokoro/` directory used by coaching state.

### D3: Atomic write
Temp file + `Path.replace()` — identical to ontology store.
Prevents corruption on crash.

## Components

| Action | Path | Purpose |
|--------|------|---------|
| CREATE | `src/kokoro/clients/store.py` | Persistence functions |
| CREATE | `tests/test_client_store.py` | Test suite |
| MODIFY | `src/kokoro/clients/__init__.py` | Export store functions |

## Interface

```python
def load_registry(project_dir: Path) -> ClientRegistry | None: ...
def save_registry(project_dir: Path, registry: ClientRegistry) -> None: ...
def create_empty_registry() -> ClientRegistry: ...
```
