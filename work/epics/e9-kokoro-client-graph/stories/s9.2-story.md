# Story: Client persistence layer

## User Story
As Kokoro,
I want to load and save client profiles from a central clients.json file,
so that client knowledge persists across sessions.

## Acceptance Criteria

### Scenario: Save and load registry
```gherkin
Given a ClientRegistry with 2 clients
When saved to clients.json
Then loading from the same path returns identical data
```

### Scenario: First run (no file)
```gherkin
Given no clients.json exists
When load_registry is called
Then it returns None (not an error)
```

### Scenario: Atomic write
```gherkin
Given a registry being saved
When the write happens
Then it uses temp file + rename (atomic, no corruption on crash)
```

## Notes
- Follow pattern from src/kokoro/ontology/store.py (atomic write, load/save/create_empty)
- File location: .kokoro/clients.json in project root
- Default path configurable but defaults to project root
