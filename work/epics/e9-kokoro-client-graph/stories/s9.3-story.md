# Story: Skill /kokoro-client

## User Story
As Eduardo,
I want a /kokoro-client skill to create, list, view, and search my clients,
so that Kokoro knows who my clients are and can recall their context.

## Acceptance Criteria

### Scenario: Create a new client
```gherkin
Given no client "Konecta Park" exists
When I run /kokoro-client and choose to create
Then a ClientProfile is created and saved to clients.json
```

### Scenario: List all clients
```gherkin
Given 2 clients exist (Konecta Park, Escuela Libre)
When I run /kokoro-client and choose to list
Then both clients are shown with group and segments
```

### Scenario: View client details
```gherkin
Given Konecta Park exists with repos, segments, metadata
When I ask to view "Konecta Park"
Then full profile is displayed including all connections
```

### Scenario: Search by segment
```gherkin
Given clients with different segments
When I search for "brokers"
Then only clients serving brokers are returned
```
