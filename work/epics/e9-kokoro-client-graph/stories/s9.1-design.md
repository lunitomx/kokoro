# S9.1 Design — Client Data Models

## Decision Summary

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Separate module `src/kokoro/clients/` | Client graph models Eduardo's relationship WITH clients, not coaching concepts. Different domain from ontology. |
| D2 | ClientProfile as Pydantic BaseModel | Same pattern as ontology/models.py. Typed fields, validation at boundaries. |
| D3 | ClientRegistry as container with search methods | find_by_id, find_by_name (case-insensitive substring), find_by_segment, list_groups. Flat list — no graph edges needed yet. |
| D4 | No EdgeType/NodeType enums | ClientProfile is flat with list fields (repos, segments). No typed relationships between clients. |

## Components

### CREATE: `src/kokoro/clients/__init__.py`

Module init with public exports (ClientProfile, ClientRegistry).

### CREATE: `src/kokoro/clients/models.py`

Two Pydantic models:

**ClientProfile** — A client node in Eduardo's knowledge graph.
- Required: `id`, `name`, `group`, `created`, `updated`
- Optional with defaults: `description`, `repos`, `campaign_folder`, `context_file`, `segments`, `industry`, `coaching_state_path`, `metadata`

**ClientRegistry** — Container for all client profiles.
- Fields: `version`, `clients`, `created`, `updated`
- Methods: `find_by_id()`, `find_by_name()`, `find_by_segment()`, `list_groups()`
- Search semantics: case-insensitive substring match for name, exact match for id, lowercase match for segment

### CREATE: `tests/test_client_models.py`

Test classes:
- `TestClientProfile`: minimal creation, full creation, missing required field validation
- `TestClientRegistry`: find_by_id, find_by_name (case-insensitive), find_by_segment, list_groups, empty registry

## Pattern Reference

Follows `src/kokoro/ontology/models.py`:
- `from __future__ import annotations`
- Pydantic BaseModel with typed fields
- `dict[str, Any]` for flexible metadata
- `list[str]` for collections (not separate models)
- datetime fields for audit trail

## Test Strategy

- Unit tests only (no I/O, no fixtures beyond in-memory objects)
- Pydantic ValidationError for missing required fields
- Search methods tested with populated and empty registries
