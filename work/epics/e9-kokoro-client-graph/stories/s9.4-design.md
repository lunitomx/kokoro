# S9.4 Design — Integración con skills existentes

## Decisions

### D1: Add standard "Client Resolution" section to kokoro-ads
kokoro-ads is the proof of concept. Other skills get the same treatment
in future stories.

### D2: Section placement — BEFORE exercises, AFTER contexto previo
The client resolution section goes between "Contexto previo" and
"Instrucciones para la sesion". Rationale: resolve WHO the client is
first, then do the work.

### D3: Use find_by_name for fuzzy matching
ClientRegistry.find_by_name does case-insensitive partial matching.
Good enough for skill resolution.

### D4: Backward compatible — no client graph = no change in behavior
If .kokoro/clients.json does not exist, the skill behaves exactly as
before. No breaking changes.

## Components

| Component | Action | Notes |
|-----------|--------|-------|
| `.claude/commands/kokoro-ads.md` | MODIFY | Add client resolution section |
| `extension/.claude/commands/kokoro-ads.md` | MODIFY | Mirror of above |

## Integration Points

- Reads `.kokoro/clients.json` via `find_by_name` (from `src/kokoro/clients/models.py`)
- Reads `context_file`, `repos`, `segments`, `metadata` from matched `ClientProfile`
- Falls back gracefully: no registry = continue without client context
- Suggests `/kokoro-client` when client not found
