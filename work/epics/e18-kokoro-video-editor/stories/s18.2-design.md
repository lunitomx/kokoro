# S18.2 Design — /kokoro-cuts

## Problem

Eduardo has video transcripts from /kokoro-listen but no automated way to
identify the best 30-60s moments for shorts and reels. He currently watches
videos manually to find cut-worthy segments — hours of work for each video.

## Value

Reduces hours of manual video review to minutes. Produces ranked, ready-to-cut
segments with suggested titles and hooks. The output feeds directly into
/kokoro-shorts (S18.3) for automated extraction.

## Approach

/kokoro-cuts is a Claude Code skill (`.claude/commands/kokoro-cuts.md`) that:

1. Reads a kokoro-listen transcript file (plain text with header metadata)
2. Uses Claude's inference to analyze content density, impactful phrases,
   topic transitions, and emotional peaks
3. Produces 1-10 ranked cuts with position references, suggested titles,
   hook text, and quality scores
4. Presents cuts to Eduardo for approval/adjustment
5. Saves structured JSON output for downstream /kokoro-shorts consumption

### Transcript Format (Input)

kokoro-listen produces plain text transcripts with this structure:

```
============================
TRANSCRIPCION
Fuente: {url}
Titulo: {title}
Duracion: {duration}
Fecha: {date}
Idioma: {lang}
Costo: ~${cost} USD
============================

{plain text transcription — no timestamps, no paragraph breaks}
```

Key constraint: Whisper is called with `response_format=text`, producing a
continuous text block without word-level timestamps. The skill must work
with this format as-is.

### Position Reference Strategy (D1)

Since transcripts lack timestamps, cuts reference positions by **sentence
boundaries** — quoting the first and last sentences of the segment. This is:
- Human-readable (Eduardo can find the segment by searching the text)
- Machine-usable (/kokoro-shorts can locate the excerpt in the transcript)
- Future-compatible (when kokoro-listen adds timestamps, the interface
  stays the same, just gains precision)

### Scoring Strategy (D2)

Pure LLM inference. Claude analyzes the full transcript in context and
identifies moments based on:
- **Content density**: high information-to-word ratio
- **Hook potential**: opening lines that grab attention
- **Topic transitions**: natural segment boundaries
- **Emotional peaks**: passion, emphasis, conviction in the text
- **Standalone value**: segment makes sense without surrounding context

No heuristic rules. The transcript is already in Claude's context window
during skill execution — leveraging Claude's language understanding is
simpler and more effective than keyword-based scoring.

### Output Format (D3)

**Human-readable** (presented to Eduardo):

```
## Cortes identificados — {video_title}

| # | Score | Titulo sugerido | Duracion est. |
|---|-------|----------------|---------------|
| 1 | 9/10  | {title}        | ~45s          |
| 2 | 8/10  | {title}        | ~35s          |

### Corte 1 — {title} (9/10)

**Hook:** "{hook text}"

**Extracto:**
> {quoted text of the segment}

**Por que este corte:** {1-line rationale}

---

### Siguiente paso

1. Aprueba los cortes que quieras convertir en shorts
2. `/kokoro-shorts` para extraer los segmentos de video
3. `/kokoro-creative` para generar thumbnails
```

**Machine-readable** (JSON saved to disk):

```json
{
  "source_transcript": "/path/to/transcript.txt",
  "video_title": "...",
  "video_duration": "3:43",
  "video_url": "...",
  "cuts_count": 2,
  "cuts": [
    {
      "rank": 1,
      "score": 9,
      "title": "Suggested short title",
      "hook": "Opening hook text for overlay",
      "excerpt": "Full text of the segment",
      "start_sentence": "First sentence of the cut...",
      "end_sentence": "Last sentence of the cut...",
      "estimated_duration_seconds": 45,
      "rationale": "Why this moment was selected"
    }
  ]
}
```

The JSON file is saved alongside the transcript:
- If transcript is at `/tmp/kokoro-listen/abc-slug.txt`
- JSON saved at `/tmp/kokoro-listen/abc-slug-cuts.json`

### Short Video Handling

For videos under 5 minutes (like the test transcript at 3:43):
- Produce 1-3 cuts instead of 5-10
- The entire video may be one viable short — that is a valid output
- Do not pad with low-quality cuts to reach a minimum count

For videos 5-30 minutes: 3-7 cuts
For videos 30+ minutes: 5-10 cuts

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Sentence-boundary references, not timestamps | Transcripts are plain text (no timestamps). Sentence refs are human-readable and machine-locatable. Future-compatible with timestamp upgrade. |
| D2 | Pure LLM inference (no heuristics) | Transcript is in Claude's context. Semantic understanding > keyword rules. Confirmed by S18.1 OpenShorts evaluation. |
| D3 | JSON with cuts array alongside transcript | /kokoro-shorts needs excerpt, title, hook, score. JSON saved next to transcript for traceability. |

## Acceptance Criteria

- **AC1**: Skill reads any kokoro-listen transcript and produces ranked cuts
- **AC2**: Each cut includes: sentence-boundary position, suggested title, hook text, quality score (1-10)
- **AC3**: Output includes human-readable markdown AND structured JSON for /kokoro-shorts
- **AC4**: Short videos (<5 min) produce 1-3 cuts; longer videos produce 5-10
- **AC5**: Cuts prioritize content density and hook-worthy opening lines
- **AC6**: Eduardo can approve/reject/adjust cuts interactively

## Must Not

- Must not require word-level timestamps (current transcripts are plain text)
- Must not call external APIs directly (uses Claude inference via skill execution)
- Must not modify kokoro-listen's output format
- Must not produce empty or zero cuts for any valid transcript
