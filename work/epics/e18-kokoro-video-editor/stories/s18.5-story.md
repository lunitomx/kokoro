# S18.5 — /kokoro-render

## User Story

**As** Eduardo (content creator),
**I want** to assemble a final video by concatenating intro, main content, and outro with normalized audio,
**so that** the output is a branded, publish-ready MP4 that plays consistently across all platforms.

## Acceptance Criteria

```gherkin
Feature: kokoro-render assembles final video with intro/outro and normalized audio

  Scenario: Concatenate intro + main video + outro
    Given an intro template video, a main content video, and an outro template video
    When I run /kokoro-render with the main video path
    Then the output video starts with the intro, followed by the main content, followed by the outro
    And transitions between segments are clean (no artifacts, no silence gaps)

  Scenario: Render without intro or outro
    Given a main content video and no intro/outro specified
    When I run /kokoro-render with --no-intro or --no-outro flags
    Then only the specified segments are included in the output
    And the output is still a valid, normalized MP4

  Scenario: Audio normalization across segments
    Given segments with different audio levels (intro at -20 LUFS, main at -14 LUFS)
    When kokoro-render processes the final assembly
    Then the output audio is normalized to a consistent level (target -16 LUFS)
    And there are no abrupt volume changes between segments

  Scenario: Export as MP4 H.264 + AAC
    Given assembled video segments
    When kokoro-render exports the final video
    Then the output is MP4 with H.264 video codec and AAC audio codec
    And the output is playable on web, mobile, and desktop without transcoding

  Scenario: Template system for intro/outro assets
    Given a client directory with an assets/templates/ folder
    When kokoro-render looks for intro/outro templates
    Then it discovers available templates from the conventional path
    And the user can specify a custom template path as override

  Scenario: Compose overlays with base video
    Given a base video and overlay assets (captions already burned in by kokoro-overlay)
    When kokoro-render assembles the final video
    Then the overlays from previous pipeline steps are preserved
    And no re-encoding degrades the caption quality

  Scenario: Output file naming and metadata
    Given a main video named "workshop-short-01-captioned.mp4"
    When kokoro-render completes
    Then the output is saved as "workshop-short-01-final.mp4"
    And a summary JSON includes output path, duration, segments used, and codec info
```

## Dependencies

- `kokoro-overlay` (S18.4, done) — provides captioned video files
- `ffmpeg` — installed on Eduardo's machine (concat demuxer, loudnorm filter)
- Intro/outro template assets (MP4 files to be provided or created)
- Test video available from S18.4 work
