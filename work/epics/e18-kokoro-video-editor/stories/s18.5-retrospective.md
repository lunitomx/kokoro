# Retrospective: S18.5 — /kokoro-render

## Summary

S18.5 (M story) created the `/kokoro-render` skill for assembling final videos
from intro, body, and outro segments using ffmpeg concat filter with two-pass
loudnorm normalization. Deliverable: `.claude/commands/kokoro-render.md` (skill
file only — D4 decision to skip knowledge file proved correct).

## What Went Well

- **D4 (no knowledge file) worked well:** The skill is leaner than S18.4's
  two-file approach. All ffmpeg concat and loudnorm instructions fit naturally
  in the single command file without duplication concerns.
- **Dynamic concat filter:** The `n=` parameter adjusts for 1-3 inputs
  (intro+body, body+outro, intro+body+outro), and all three variants are
  documented in the skill.
- **Two-pass loudnorm:** Correct default for speech content normalization —
  first pass measures, second pass applies, avoiding the artifacts of
  single-pass loudnorm.
- **Template discovery priority:** Explicit args always override convention-based
  file discovery, which prevents surprising behavior when users have both.

## What to Improve

- **QR caught missing variants:** The intro-only variant and priority inversion
  between explicit args and convention were caught by quality review. Both would
  have been caught earlier if design examples covered all 3 partial input
  combinations explicitly.
- **Design should enumerate all dynamic variants:** When a design specifies
  "dynamic" behavior (like variable input counts), every variant needs a
  concrete example in the design doc — not just the happy path.
