### T2: GREEN — Create skill file, knowledge file, and extension mirrors

Create `.claude/commands/kokoro-shorts.md` with the full 6-step process:

1. **Read cuts JSON** — validate structure (cuts array, video_url,
   start_sentence/end_sentence per cut). Accept optional manual
   `start_time`/`end_time` overrides per cut.
2. **Locate source video** — check `video_url` from cuts JSON. Look for
   existing audio at `/tmp/kokoro-listen/{video_id}.mp3` (reuse
   kokoro-listen download). If not found, download via yt-dlp.
3. **Re-transcribe with Whisper verbose_json** — call Whisper API with
   `response_format=verbose_json` to get segment-level timestamps.
   Skip if all cuts have manual timestamp overrides.
4. **Fuzzy-match sentence boundaries** — for each cut, match
   `start_sentence` against Whisper segments (normalize: lowercase,
   strip punctuation, collapse whitespace; substring containment first,
   then longest common subsequence ratio >= 0.7). Report errors for
   unresolvable cuts and continue with remaining.
5. **ffmpeg extraction** — two-pass per cut:
   - Pass 1: stream copy cut (`-ss {start} -to {end} -i {input} -c copy`)
   - Pass 2: re-encode with fades (`fade=t=in:st=0:d=0.5`,
     `fade=t=out:st={dur-0.5}:d=0.5`, H.264/AAC)
   - Optional 9:16 vertical reformat (center crop `ih*9/16:ih`,
     scale 1080x1920) when user requests "vertical" or "9:16"
6. **Save output** — shorts as `{basename}-short-01.mp4`, etc.
   Summary JSON (`{basename}-shorts.json`) with per-short status
   (success/error), paths, durations, resolution method.

Output template with: summary table (rank, file, duration, status),

## Risks

| Risk | Mitigation |
|------|-----------|
| Whisper word-level alignment drift on long videos | Sentence-level fuzzy matching with 0.7 threshold; fallback to segment boundaries |
| ffmpeg keyframe seeking inaccuracy | Use `-ss` before `-i` for speed; fall back to after `-i` if tolerance exceeded |
| Sentence text mismatch (punctuation, filler words) | Normalize both strings; substring containment fast path before fuzzy ratio |
| Large video processing time | Acceptable for M story; two-pass avoids full re-encode |
