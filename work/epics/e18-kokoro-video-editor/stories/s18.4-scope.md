# Scope: S18.4 — /kokoro-overlay

## In Scope

- Synchronized captions/subtitles from transcript timestamps on video shorts
- ffmpeg drawtext filter or ASS subtitle rendering (decide during design)
- Vertical 9:16 (1080x1920) as the DEFAULT output format (lesson from M1)
- Consistent visual style: font, color, size, stroke/shadow for readability
- Caption positioning in lower third of frame
- Word-level or phrase-level caption timing (from Whisper verbose_json)
- Output naming convention: {basename}-captioned.mp4
- Summary JSON with output path, caption count, and metadata
- Skill file at .claude/commands/kokoro-overlay.md
- Tests with real transcript and mock/real ffmpeg commands

## Out of Scope

- Animated text effects (kinetic typography) — too complex for M story
- Custom font uploads or per-video style configuration
- Intro/outro attachment (that is S18.5 kokoro-render)
- Audio normalization or background music
- Upload to any platform (YouTube, TikTok, Instagram)
- Speaker diarization (different colors per speaker)
- Burned-in graphics or logos (that is S18.5 territory)
- Auto-translation of captions to other languages

## Core Technical Decisions (to resolve in design)

1. **drawtext vs ASS subtitles:** drawtext is simpler (single ffmpeg filter),
   ASS gives more style control (fonts, colors, positioning, animations).
   Decide based on complexity vs readability tradeoff.
2. **Word-level vs phrase-level captions:** Word-level (karaoke-style) is trendy
   but complex. Phrase-level (2-4 words at a time) is simpler and proven.
   Whisper verbose_json provides both options.
3. **Vertical reformat integration:** Should overlay handle vertical reformat
   (absorbing that responsibility from kokoro-shorts) or expect pre-formatted
   input? Default to 9:16 output regardless — if input is horizontal, reformat first.

## Done Criteria

- [ ] Skill file created at `.claude/commands/kokoro-overlay.md`
- [ ] Captions are synchronized with audio (within 0.5s tolerance)
- [ ] Output defaults to vertical 9:16 (1080x1920) format
- [ ] Visual style is consistent and readable (stroke/shadow on text)
- [ ] Caption positioning in lower third, no visual overlap
- [ ] Output file follows naming convention and is playable
- [ ] Summary JSON saved with paths, caption count, and metadata
- [ ] All tests pass (unit + integration)
- [ ] Type checks pass (pyright strict)

## Risks

- Word-level timestamp accuracy from Whisper may drift on fast speech — mitigate
  with phrase-level grouping (2-4 words) instead of single-word display
- ffmpeg drawtext escaping is fragile (special characters in transcript) — mitigate
  with proper escaping function and tests
- Font availability across systems — mitigate by using system fonts or bundling
  a default font file
- Long captions may overflow frame width — mitigate with word wrapping or
  max character count per line
- ASS subtitle format has a learning curve — mitigate by starting with drawtext
  and upgrading if needed
