### T2: GREEN — Create skill file

**Goal:** All tests pass. The skill file exists with complete content.

**Steps:**
1. Create `.claude/commands/kokoro-render.md` with full skill instructions:
   - **Template Discovery (D3):** Convention-based path search — per-invitado (`clientes/{invitado}/assets/intro.mp4` / `outro.mp4`), then project default (`assets/templates/intro.mp4` / `outro.mp4`), then user-provided `--intro` / `--outro` paths. `--no-intro` / `--no-outro` flags to skip.
   - **Audio Normalization (D2):** Two-pass loudnorm at -16 LUFS. Pass 1 measures (`loudnorm=I=-16:TP=-1.5:LRA=11:print_format=json`), parse JSON for measured values. Pass 2 applies with `linear=true`, `-c:v copy` (no video re-encode during normalization).
   - **Concatenation (D1):** Concat filter with re-encode. Dynamic segment count (intro + main + outro, or subset). `-c:v libx264 -preset fast -crf 23 -c:a aac -b:a 128k`. Handles mismatched codecs/resolutions between templates and main video.
   - **Output:** `{basename}-final.mp4` in same directory as input. Summary JSON with source, output, templates used, loudnorm measurements, duration, segments, codec info.
   - **HITL gate:** Display each ffmpeg command before execution, wait for confirmation.
   - **Error handling:** ffmpeg not installed detection, template not found logging, loudnorm parse failure reporting, concat filter failure with ffmpeg stderr.
   - **Graceful degradation:** If no templates found, normalize + export main video directly as `{basename}-final.mp4`.
2. No separate knowledge file (D4 — learned from S18.4 AR: simpler commands fit in one file).
3. Run `uv run pytest tests/test_output_structure.py -x` — expect PASS
4. Run `uv run pytest tests/ -x` — full suite pass
5. Commit (GREEN state)

**Verification:** All tests pass. Skill file exists and is well-formed markdown. Cross-references resolve to valid commands.

**Files created:**
- `.claude/commands/kokoro-render.md`
