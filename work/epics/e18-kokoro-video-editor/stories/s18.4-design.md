# S18.4 Design — /kokoro-overlay

## Problem

Shorts extracted by /kokoro-shorts lack captions. 80%+ of social media viewers
watch on mute, making uncaptioned shorts ineffective for Reels, Shorts, and
TikTok. The M1 milestone hallazgo confirmed shorts must be vertical AND
captioned to be publish-ready.

## Value

Eduardo's shorts become publish-ready with synchronized captions, increasing
engagement and accessibility. Completes the most valuable piece of Phase 2
(M2: Contenido Pulido). A short that works on mute reaches 4x more viewers
than one that requires audio.

## Approach

/kokoro-overlay is a Claude Code skill (`.claude/commands/kokoro-overlay.md`)
that takes a short video + Whisper verbose_json transcript and produces a
vertical 9:16 video with synchronized drawtext captions.

### Pipeline: One ffmpeg Command

The skill builds a single ffmpeg command that handles three operations in one
pass: vertical reformat (crop + scale), caption overlay (drawtext filter chain),
and encoding (H.264 + AAC).

```bash
ffmpeg -y -i "input-short.mp4" \
  -vf "crop=ih*9/16:ih,scale=1080:1920,\
drawtext=text='La riqueza no se mide en pesos':fontsize=48:fontcolor=white:borderw=3:bordercolor=black:x=(w-text_w)/2:y=h*0.80:enable='between(t,0.0,3.2)',\
drawtext=text='se mide en la capacidad de crear valor':fontsize=48:fontcolor=white:borderw=3:bordercolor=black:x=(w-text_w)/2:y=h*0.80:enable='between(t,3.2,6.8)'" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a aac -b:a 128k \
  "input-short-captioned.mp4"
```

### Stage 1: Transcript Segment Extraction

The skill receives two inputs:
1. A short video file (from /kokoro-shorts output)
2. A Whisper verbose_json file (from /kokoro-listen or /kokoro-shorts)

From the Whisper JSON, it extracts segments that fall within the short's time
range. Each segment provides `start`, `end`, and `text` fields — these become
caption entries directly.

**Input format (Whisper verbose_json segments):**

```json
{
  "segments": [
    {"id": 0, "start": 0.0, "end": 3.2, "text": "La riqueza no se mide en pesos"},
    {"id": 1, "start": 3.2, "end": 6.8, "text": "se mide en la capacidad de crear valor"},
    {"id": 2, "start": 6.8, "end": 10.5, "text": "para las personas que te rodean"},
    {"id": 3, "start": 10.5, "end": 14.0, "text": "Eso es lo que un guardian protege"}
  ]
}
```

If a shorts summary JSON from /kokoro-shorts is provided instead, the skill
reads the `start_time` and `end_time` for each short and filters Whisper
segments accordingly, adjusting timestamps to be relative to the short's start.

### Stage 2: Text Escaping (D5)

ffmpeg drawtext has fragile special character handling. The skill applies
escaping before building the filter:

```python
def escape_drawtext(text: str) -> str:
    text = text.replace("\\", "\\\\")
    text = text.replace("'", "'\\''")
    text = text.replace(":", "\\:")
    text = text.replace("%", "%%")
    return text
```

Characters escaped: `\`, `'`, `:`, `%`. These are the drawtext metacharacters
that cause ffmpeg filter parsing failures.

### Stage 3: Line Wrapping

If a segment exceeds ~35 characters (for 1080px width at fontsize 48), split
into 2 lines at the nearest word boundary:

```
"se mide en la capacidad de crear valor"
→ "se mide en la capacidad\nde crear valor"
```

This prevents text overflow beyond frame width. The 35-char threshold is
approximate — optimized for Spanish text at fontsize 48 on 1080px width.

### Stage 4: ffmpeg Filter Construction (D1)

Build a drawtext filter chain. Each caption segment becomes one drawtext
filter with `enable='between(t,start,end)'` for timing.

Caption style:
- `fontsize=48` — readable on mobile
- `fontcolor=white` — high contrast
- `borderw=3` + `bordercolor=black` — readable against any background
- `x=(w-text_w)/2` — horizontally centered
- `y=h*0.80` — lower third positioning (80% from top)

### Stage 5: Vertical Reformat (D3)

Default output is 9:16 (1080x1920). The crop+scale happens in the same
filter chain as captions:

```
crop=ih*9/16:ih,scale=1080:1920,drawtext=...,drawtext=...
```

Order matters: crop and scale FIRST, then drawtext. The caption positioning
(`y=h*0.80`) references the final frame dimensions (1920px height), so
reformat must precede overlay.

User can override to keep original aspect ratio by saying "keep original" or
"no reformat" in the invocation message.

### Stage 6: Output

- Video: `{basename}-captioned.mp4` in same directory as input
- Summary JSON: `{basename}-captioned.json`

**Summary JSON format:**

```json
{
  "source_video": "/tmp/kokoro-shorts/workshop-short-01.mp4",
  "output_video": "/tmp/kokoro-shorts/workshop-short-01-captioned.mp4",
  "format": "vertical_9_16",
  "resolution": "1080x1920",
  "captions_count": 4,
  "duration_seconds": 14.0,
  "style": {
    "font_size": 48,
    "font_color": "white",
    "border_width": 3,
    "border_color": "black",
    "position": "lower_80pct"
  }
}
```

### Invocation

```
/kokoro-overlay /tmp/kokoro-shorts/workshop-short-01.mp4 /tmp/kokoro-shorts/workshop-whisper.json
```

Optional instructions via message:
- "keep original" or "no reformat" — skip vertical crop+scale
- "batch" or multiple files — process all shorts from a shorts summary JSON

### Batch Mode

When given a shorts summary JSON (from /kokoro-shorts), process all successful
shorts in sequence:

```
/kokoro-overlay /tmp/kokoro-shorts/workshop-shorts.json /tmp/kokoro-shorts/workshop-whisper.json
```

The skill reads the shorts JSON, filters Whisper segments for each short's
time range, adjusts timestamps to be relative, and runs one ffmpeg command
per short.

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | drawtext (not ASS subtitles) | Simpler, single ffmpeg command, no external .ass file to manage. Sufficient for phrase-level captions. ASS would add complexity (file generation, font embedding) for features we don't need (animations, multi-color, karaoke). |
| D2 | Phrase-level captions (not word-level) | Whisper verbose_json provides segment-level timestamps natively (2-8 words per segment). Proven effective on social media. Word-level requires additional API params AND complex drawtext chaining per word — excessive for M story. |
| D3 | Overlay handles vertical reformat (default 9:16) | The overlay must know final frame dimensions to position captions correctly. One ffmpeg command does crop+scale+drawtext. If reformat happened separately, it adds a pipeline step and the caption positioning would depend on knowing whether reformat was applied. |
| D4 | No re-transcription — expects existing Whisper JSON | kokoro-shorts already produces/uses Whisper verbose_json. Re-transcribing would waste ~$0.02 and duplicate work. The transcript is already available as a pipeline artifact. |
| D5 | Explicit escaping function for drawtext special chars | ffmpeg drawtext parsing is fragile — unescaped `:`, `'`, `\`, or `%` cause silent failures or wrong output. Explicit escaping with a documented function is more reliable than hoping the text is clean. |

## Acceptance Criteria

- **AC1**: Captions appear synchronized with audio (within 0.5s of Whisper timestamps)
- **AC2**: Output defaults to vertical 9:16 (1080x1920); user can override to keep original
- **AC3**: Captions readable against any background (white text, black stroke border width 3)
- **AC4**: Output saves as `{basename}-captioned.mp4` with summary JSON
- **AC5**: Special characters in transcript are properly escaped for ffmpeg drawtext

## Should

- Long lines wrap at ~35 chars to prevent overflow on 1080px width
- Fade in/out on final output (0.5s, consistent with kokoro-shorts)

## Must Not

- Must not require external font files (use ffmpeg default or system fonts)
- Must not re-transcribe (expects existing Whisper verbose_json as input)
- Must not upload or publish anywhere
- Must not modify the input video or transcript files
- Must not fail silently — all errors visible in summary JSON or reported clearly

## Error Handling

- **Missing Whisper JSON**: report error, suggest running /kokoro-listen first
- **No segments in time range**: report empty captions, produce video without captions (with warning)
- **ffmpeg not installed**: detect early, abort with clear message
- **Drawtext escaping failure**: log the problematic text, skip that caption, continue with others
- **Input video not found**: report path and abort
