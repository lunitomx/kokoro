# S18.5 Design — /kokoro-render

## Problem

Captioned shorts from /kokoro-overlay lack professional polish — no branding
(intro/outro) and inconsistent audio levels between segments. A short that
looks amateur gets scrolled past; consistent branding and normalized audio
signal quality and build recognition.

## Value

Eduardo's videos get a professional finish with consistent branding and audio,
making them broadcast-ready without manual editing. Completes M2 (Contenido
Pulido) alongside S18.4.

## Approach

/kokoro-render is a Claude Code skill (`.claude/commands/kokoro-render.md`)
that takes a video (typically captioned output from /kokoro-overlay), discovers
or receives intro/outro templates, normalizes audio via two-pass loudnorm,
concatenates intro + main + outro via the concat filter, and exports a final
MP4.

### Stage 1: Template Discovery (D3)

The skill searches for intro/outro templates using convention-based paths,
in priority order:

1. `clientes/{invitado}/assets/intro.mp4` / `outro.mp4` (per-invitado)
2. `assets/templates/intro.mp4` / `outro.mp4` (project default)
3. User-provided paths via `--intro` / `--outro` arguments

If no templates are found, the skill proceeds without them (audio normalization
+ final export only). The `--no-intro` and `--no-outro` flags explicitly skip
template discovery for that segment.

### Stage 2: Audio Normalization — Two-Pass Loudnorm (D2)

Two-pass loudnorm for accurate normalization. Target: -16 LUFS (YouTube/social
media standard).

**Pass 1 — Measure:**

```bash
ffmpeg -i "{input}" -af loudnorm=I=-16:TP=-1.5:LRA=11:print_format=json -f null - 2>&1
```

Parse the JSON output to extract: `measured_I`, `measured_TP`, `measured_LRA`,
`measured_thresh`, `offset`.

**Pass 2 — Apply:**

```bash
ffmpeg -y -i "{input}" \
  -af "loudnorm=I=-16:TP=-1.5:LRA=11:measured_I={mI}:measured_TP={mTP}:measured_LRA={mLRA}:measured_thresh={mThresh}:offset={offset}:linear=true" \
  -c:v copy -c:a aac -b:a 128k \
  "{normalized}"
```

Pass 2 uses `-c:v copy` to avoid re-encoding video during normalization.
The video re-encode happens only once, during the concat stage.

If no intro/outro templates exist (normalization-only mode), pass 2 produces
the final output directly as `{basename}-final.mp4`.

### Stage 3: Concatenation via Concat Filter (D1)

When intro and/or outro templates exist, concatenate using the concat filter
with re-encode:

```bash
# Full: intro + main + outro
ffmpeg -y -i "{intro}" -i "{normalized_main}" -i "{outro}" \
  -filter_complex "[0:v][0:a][1:v][1:a][2:v][2:a]concat=n=3:v=1:a=1[outv][outa]" \
  -map "[outv]" -map "[outa]" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a aac -b:a 128k \
  "{output}"

# Partial: main + outro (no intro)
ffmpeg -y -i "{normalized_main}" -i "{outro}" \
  -filter_complex "[0:v][0:a][1:v][1:a]concat=n=2:v=1:a=1[outv][outa]" \
  -map "[outv]" -map "[outa]" \
  -c:v libx264 -preset fast -crf 23 \
  -c:a aac -b:a 128k \
  "{output}"
```

The concat filter handles mismatched codecs/resolutions between templates and
the main video. For shorts (30-60s), re-encode time is negligible.

The number of inputs and concat `n=` parameter adjust dynamically based on
which segments are present (intro, main, outro in any combination).

### Stage 4: Output

- Video: `{basename}-final.mp4` in same directory as input
- Summary JSON: `{basename}-final.json`

**Summary JSON format:**

```json
{
  "source_video": "/tmp/kokoro-shorts/workshop-short-01-captioned.mp4",
  "output_video": "/tmp/kokoro-shorts/workshop-short-01-final.mp4",
  "intro": "assets/templates/intro.mp4",
  "outro": "assets/templates/outro.mp4",
  "audio_normalization": {
    "target_lufs": -16,
    "method": "loudnorm_two_pass",
    "measured_I": -18.3,
    "measured_TP": -2.1
  },
  "duration_seconds": 48.5,
  "segments": ["intro", "main", "outro"],
  "codec": {"video": "H.264", "audio": "AAC 128k"}
}
```

### Invocation

```
/kokoro-render /tmp/kokoro-shorts/workshop-short-01-captioned.mp4
```

Optional arguments via message:
- `--intro path/to/intro.mp4` — explicit intro template
- `--outro path/to/outro.mp4` — explicit outro template
- `--no-intro` — skip intro
- `--no-outro` — skip outro
- Invitado name for per-invitado template discovery

### HITL Gate

Before executing each ffmpeg command, display the full command and wait for
confirmation. This follows the established pattern from /kokoro-shorts and
/kokoro-overlay.

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Concat filter with re-encode (not demuxer) | Intro/outro templates may not match the main video's codec/resolution (especially after vertical reformat). The concat filter handles mismatched inputs gracefully. For shorts (30-60s), re-encode time is negligible. The demuxer requires strict format matching that's fragile and hard to debug. |
| D2 | Two-pass loudnorm (not single-pass) | Two-pass measures actual loudness first, then normalizes accurately. For shorts (30-60s), the extra pass adds <2 seconds. Single-pass can overshoot or undershoot on speech content. Target: -16 LUFS (YouTube/social media standard). |
| D3 | Convention-based template discovery | Look for intro/outro at well-known paths (per-invitado, then project default). If not found, skip gracefully. Override via explicit path argument. No config file needed — convention over configuration. |
| D4 | No separate knowledge file | Based on AR finding from S18.4 (duplication between skill and knowledge files). The ffmpeg commands for concat + loudnorm are simpler than overlay drawtext chains — fits in one file. Only create knowledge file if content grows too large. |

## Acceptance Criteria

- **AC1**: Concat intro + main + outro in correct order (or subset with --no-intro/--no-outro)
- **AC2**: Audio normalized to -16 LUFS via two-pass loudnorm
- **AC3**: Output is valid MP4 H.264 + AAC playable on all platforms
- **AC4**: Template discovery from convention paths works
- **AC5**: Output saved as {basename}-final.mp4 with summary JSON

## Should

- Graceful degradation if no templates found (just normalize + export main video)
- Show ffmpeg command before executing (HITL confirmation)

## Must Not

- Must not modify or re-encode captions burned in by kokoro-overlay
- Must not require templates to exist (skip gracefully)
- Must not upload or publish anywhere
- Must not modify the input video file

## Error Handling

- **ffmpeg not installed**: detect early, abort with clear message
- **Template not found at convention paths**: log which paths were checked, proceed without templates
- **loudnorm parse failure**: report raw ffmpeg output, abort normalization
- **Concat filter failure**: report ffmpeg stderr, suggest checking template format compatibility
- **Input video not found**: report path and abort
