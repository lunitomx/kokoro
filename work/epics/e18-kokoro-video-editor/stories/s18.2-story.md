# S18.2 — /kokoro-cuts

## User Story

**As** Eduardo (content creator),
**I want** to receive a ranked list of the best moments from a video transcript,
**so that** I can quickly select which segments to turn into shorts without watching the full video again.

## Acceptance Criteria

```gherkin
Feature: kokoro-cuts identifies best moments from transcript

  Scenario: Analyze transcript and propose cuts
    Given a transcript file from kokoro-listen with timestamps
    When I run /kokoro-cuts with the transcript path
    Then I receive 5-10 proposed cuts ranked by quality
    And each cut includes start timestamp, end timestamp, suggested title, and hook
    And each cut duration is between 30 and 60 seconds

  Scenario: Cut analysis covers multiple signal types
    Given a transcript with varied content
    When kokoro-cuts analyzes the transcript
    Then cuts are scored based on content density, impactful phrases, topic changes, and high-energy moments
    And the ranking reflects a combination of these signals

  Scenario: Eduardo reviews and adjusts cuts
    Given a list of proposed cuts
    When Eduardo reviews the output
    Then he can see each cut with its context (surrounding text)
    And the output format supports easy selection/discard decisions

  Scenario: Short transcript produces fewer cuts
    Given a transcript shorter than 5 minutes
    When kokoro-cuts analyzes it
    Then it produces at least 1 cut and at most proportional to duration
    And it does not pad with low-quality cuts

  Scenario: Output is machine-readable for kokoro-shorts
    Given kokoro-cuts has produced a cuts list
    When the output is passed to kokoro-shorts
    Then the timestamps and metadata are parseable without transformation
```
