# Scope: S18.5 — /kokoro-render

## In Scope

- Concatenate intro + main video + outro using ffmpeg concat demuxer
- Audio normalization via ffmpeg loudnorm filter (target -16 LUFS, two-pass)
- Export as MP4 H.264 + AAC for universal compatibility
- Template system: discover intro/outro from conventional path (assets/templates/)
- Custom template path override via skill arguments
- Optional flags to skip intro or outro (--no-intro, --no-outro)
- Preserve quality from upstream pipeline (kokoro-overlay captioned output)
- Output naming convention: {basename}-final.mp4
- Summary JSON with output path, duration, segments, codec info
- Skill file at .claude/commands/kokoro-render.md
- Tests with real ffmpeg commands and mock assets

## Out of Scope

- Creating intro/outro video assets from scratch (templates are pre-made assets)
- Background music mixing or multi-track audio editing
- Animated transitions between segments (hard cuts are sufficient)
- Upload to any platform (YouTube, TikTok, Instagram)
- Thumbnail generation (could be a future story)
- Batch rendering of multiple videos in parallel
- Re-encoding or modifying the captions burned in by kokoro-overlay
- Custom codec profiles or format variants beyond MP4 H.264

## Core Technical Decisions (to resolve in design)

1. **Concat method:** ffmpeg concat demuxer (file list) vs concat filter.
   Demuxer is faster (no re-encode) but requires matching codecs/resolution.
   Filter is slower but handles mismatched inputs. Decide based on whether
   intro/outro templates will always match the main video codec/resolution.
2. **Loudnorm two-pass vs single-pass:** Two-pass gives better results
   (measures first, normalizes second) but doubles processing time.
   For shorts (30-60s), two-pass is fast enough. For longer videos, evaluate.
3. **Template discovery:** Convention-based (assets/templates/intro.mp4,
   assets/templates/outro.mp4) vs explicit config file. Start with convention,
   add config if needed.

## Risks

- Intro/outro templates may have different codec/resolution than main video —
  mitigate by standardizing templates or using concat filter with re-encode
- loudnorm two-pass increases processing time on long videos — mitigate by
  targeting shorts (30-60s) where two-pass is negligible
- Concat demuxer requires matching stream formats — mitigate by detecting
  mismatches and falling back to concat filter with re-encode
- Template assets not yet created — mitigate by providing sensible defaults
  and documenting the expected format for Eduardo to create templates
