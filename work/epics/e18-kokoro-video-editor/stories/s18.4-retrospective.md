# Retrospective: S18.4 — /kokoro-overlay

## Summary

S18.4 (M story) created the `/kokoro-overlay` skill and its companion knowledge
file for adding synchronized captions to video shorts using ffmpeg drawtext.
Deliverables: `.claude/commands/kokoro-overlay.md` (skill file) and
`.claude/knowledge/kokoro-overlay-ffmpeg.md` (drawtext reference/troubleshooting).

## What Went Well

- **Clean TDD cycle:** RED test added to EXISTING_COMMANDS, GREEN with skill +
  knowledge file creation, refactor minimal — the pattern from S18.3 carried over
  smoothly.
- **Design decisions validated:** All five design decisions (D1: drawtext over ASS,
  D2: phrase-level captions, D3: integrated vertical reformat, D4: escaped drawtext
  chars, D5: conservative line wrap at ~35 chars) proved correct during implementation.
- **Architecture Review and Quality Review caught real issues:** AR flagged
  duplication between skill and knowledge files (a pattern-level concern across E18).
  QR caught that T3 updated the knowledge file instead of the command file as the
  plan specified.
- **Knowledge file as reference:** The ffmpeg drawtext knowledge file serves as a
  standalone troubleshooting guide — useful beyond this skill.

## What to Improve

- **More explicit file paths in plan tasks:** T3 specified updating
  `.claude/commands/kokoro-shorts.md` but the implementation agent updated the
  knowledge file instead. Including the exact file path prominently (not just in
  the "Files modified" section) would reduce this risk.
- **Reduce skill/knowledge duplication:** The escaping rules appear in both the
  skill file and the knowledge file. A convention could help: skill file =
  instructions and pipeline steps, knowledge file = reference and troubleshooting
  only. This applies to kokoro-shorts as well.

## Heutagogical Checkpoint

### 1. What did you learn?

- ffmpeg drawtext filter requires careful escaping of special characters
  (`\`, `'`, `:`, `%`) — and the ORDER of escaping matters (`\` must be escaped
  first to prevent double-escaping).
- Vertical reformat (crop+scale) must precede drawtext in the filter chain because
  caption positioning (`y=h*0.80`) references final frame dimensions.
- Phrase-level captions (Whisper segments) are simpler and more effective than
  word-level for social media — less visual noise, easier to read.

### 2. What would you change about the process?

- The quality review caught that the kokoro-shorts command file was not updated
  (only the knowledge file was). The plan specified T3 should update the command
  file, but the implementation agent updated the wrong file. More explicit file
  paths in plans would help prevent this.

### 3. Are there improvements for the framework?

- AR noted duplication between skill files and knowledge files. This is a
  pattern-level issue across all E18 skills (kokoro-shorts has the same pattern).
  A convention could be established: skill file = instructions, knowledge file =
  troubleshooting only.

### 4. What are you more capable of now?

- Building ffmpeg drawtext filter chains with timing, escaping, and combined
  crop+scale+text overlays in a single command.
- Understanding the interplay between filter chain ordering and coordinate systems
  in ffmpeg video processing.
