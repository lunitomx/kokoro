# Scope: S18.2 — /kokoro-cuts

## In Scope

- Transcript parsing: read kokoro-listen output (timestamped text segments)
- Signal analysis: content density, impactful phrases, topic transitions, high-energy moments
- Cut proposal: 5-10 ranked cuts with start/end timestamps (30-60s each)
- Metadata per cut: suggested title, hook phrase, quality score, signal breakdown
- Human-readable output for Eduardo to review and select/discard
- Machine-readable output (JSON) for downstream consumption by kokoro-shorts
- Proportional cut count for short transcripts (< 5 min)
- Test coverage with real transcript from kokoro-listen

## Out of Scope

- Video file manipulation (that is S18.3 kokoro-shorts)
- Speaker diarization or voice activity detection
- Sentiment analysis or emotion detection (keep signals simple)
- Interactive UI for cut selection (text-based output is sufficient)
- Integration with kokoro-shorts pipeline (downstream story)
- LLM-based analysis (use heuristic scoring first, LLM is Phase 2 enhancement)

## Done Criteria

- [ ] Skill file created at `.claude/skills/kokoro-cuts.md`

- [ ] Signal scoring produces ranked cuts from test transcript

- Transcript format from kokoro-listen may vary — mitigate by reading actual output first
- Heuristic scoring may not surface the "best" moments — mitigate with HITL review
- 30-60s window may split mid-sentence — mitigate with sentence-boundary snapping
