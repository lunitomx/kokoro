## Core Technical Challenge

kokoro-cuts outputs `start_sentence` and `end_sentence` — text boundaries, not timestamps.
To cut video with ffmpeg, we need actual timestamps. Resolution strategy:

1. **Primary:** Re-run Whisper with `response_format=verbose_json` on the audio track
   to get word-level timestamps. Match `start_sentence` text against words to find
   the start time; match `end_sentence` text to find the end time.
2. **Fallback:** If kokoro-listen was run with timestamps (segment-level), parse those
   segments and fuzzy-match the sentence text to find the closest segment boundaries.
3. **Manual override:** Allow Eduardo to provide approximate timestamps directly
   in the cuts JSON (optional `start_time` / `end_time` fields that bypass resolution).

## Risks

- Whisper word-level timestamps may have alignment drift on long videos — mitigate with
  sentence-level fuzzy matching as fallback
- ffmpeg keyframe seeking vs exact seeking tradeoff — use `-ss` before `-i` for speed,
  fall back to after `-i` for accuracy if needed
- Large video files may take significant processing time — acceptable for M story,
  optimization deferred
- Sentence text from cuts may not match Whisper output exactly (punctuation, filler words)
  — use fuzzy matching with high threshold
