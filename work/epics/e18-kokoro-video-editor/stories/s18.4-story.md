# S18.4 — /kokoro-overlay

## User Story

**As** Eduardo (content creator),
**I want** to generate synchronized captions/subtitles on my shorts from the transcript,
**so that** viewers can follow the content without audio (80%+ watch on mute) and the shorts are publish-ready.

## Acceptance Criteria

```gherkin
Feature: kokoro-overlay generates synchronized captions on video shorts

  Scenario: Generate captions from transcript segment
    Given a short video file produced by kokoro-shorts
    And a transcript segment with word-level or phrase-level timestamps
    When I run /kokoro-overlay with the video path and transcript segment
    Then the output video contains synchronized text captions
    And captions appear and disappear in sync with the spoken words

  Scenario: Default vertical 9:16 output format
    Given an input video (any aspect ratio)
    When kokoro-overlay renders the output
    Then the output defaults to 9:16 (1080x1920) vertical format
    And the user can override to keep original aspect ratio

  Scenario: Consistent visual style applied to captions
    Given caption text to render on video
    When ffmpeg renders the captions
    Then a consistent font, color, size, and position is applied
    And captions are positioned in the lower third of the frame
    And text is readable against any background (stroke or shadow)

  Scenario: Rendering via ffmpeg drawtext or ASS subtitles
    Given a set of timed caption entries
    When kokoro-overlay renders the video
    Then it uses ffmpeg drawtext filter or ASS subtitle format
    And the output is a valid MP4 file (H.264 + AAC)

  Scenario: Multiple caption segments are rendered sequentially
    Given a transcript with 15 caption phrases over a 45-second short
    When kokoro-overlay processes the video
    Then all 15 phrases appear at their correct timestamps
    And no captions overlap visually

  Scenario: Output file naming and metadata
    Given an input short named "workshop-short-01.mp4"
    When kokoro-overlay completes
    Then the output is saved as "workshop-short-01-captioned.mp4"
    And a summary JSON includes the output path and caption count
```

## Dependencies

- `kokoro-shorts` (S18.3, done) — provides extracted short video files
- `kokoro-listen` (existing) — provides transcript with timestamps (verbose_json for word-level)
- `ffmpeg` — installed on Eduardo's machine
- Test video and transcript available from S18.3 work
