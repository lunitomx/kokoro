## 3. How Moment Detection Works

1. **Download** video via yt-dlp
2. **Transcribe** with faster-whisper (word-level timestamps, base model, INT8 quantization)
3. **Send full transcript** to Gemini with a detailed prompt (~40 lines)
4. **Gemini returns JSON**: 3-15 viral moments with timestamps, descriptions, titles, hooks
5. Clips extracted with ffmpeg using the returned timestamps

Key characteristics:
- Pure LLM-based detection — no silence detection, no audio energy analysis
- Prompt instructs Gemini to find natural cuts at silence moments
- Target clip length: 15-60 seconds
- Results ordered by predicted performance

## 4. Face Tracking for 9:16 Reframe

### TRACK Mode

Two complementary algorithms:

**SmoothedCameraman** — "heavy tripod" panning behavior:
- Only moves when subject leaves the safe zone (25% of crop width)
- Slow pans: 3 px/frame for gradual tracking
- Fast re-frames: 15 px/frame for scene changes
- Produces smooth, professional-looking camera follow

**SpeakerTracker** — multi-face with identity matching:
- Face ID assignment with score decay over time
- Speaker switch cooldown to prevent jitter between faces
- Selects active speaker based on scoring heuristic

### GENERAL Mode

Blurred background padding: center crop + blur fill for non-face content.

**Output:** 1080x1920 (9:16, hardcoded resolution).

- **Zero tests** — no test directory, no test files, no test configuration

- **No Pydantic** — raw dicts throughout
- **Monolithic files** — app.py (2039 lines), saasshorts.py (1474 lines), main.py (1020 lines)
- **Global state** — shared mutable state across modules
- **Hardcoded prompts** — Gemini prompts embedded as string literals
- **No error handling strategy** — bare try/except or none at all
- **No linting config** — no ruff, flake8, or mypy configuration

### Positives

- Actively maintained (last push 4 days ago)
- MIT license (no legal barriers)

## 8. Verdict: INSPIRATION-ONLY

**Rationale:**

1. **Cannot integrate as dependency** — no package structure, no public API, web-app-only architecture
2. **Cannot fork productively** — 6400 lines of untyped, untested code would require more effort to rehabilitate than to rewrite
3. **Valuable algorithms are extractable** — the core logic (~200 lines each) can be studied, understood, and reimplemented with proper standards
4. **Building from scratch with our standards is faster** — TDD, typed, Pydantic-modeled code from day one avoids the debt entirely

OpenShorts validates the approach (Gemini for moment detection, face tracking for reframe, faster-whisper for transcription) but its implementation is not reusable as-is.

## 9. Reference Algorithms (for S18.3+)

These four algorithms should be studied and reimplemented with Kokoro standards:

### 9.1 Gemini Prompt for Viral Moment Detection (~40 lines)

Detailed prompt that instructs Gemini to analyze a full transcript and return structured JSON with 3-15 viral moments. Includes guidance on finding natural silence cuts, target durations, hook quality, and performance ranking. The prompt design is the main intellectual property — the surrounding code is trivial.

### 9.2 SmoothedCameraman Logic (~80 lines)

"Heavy tripod" tracking algorithm: safe zone (25% crop width), slow pan (3 px/frame), fast re-frame (15 px/frame). Produces professional camera follow behavior. Key insight: the smoothing parameters are what make it look natural vs robotic.

### 9.3 SpeakerTracker Logic (~120 lines)

Multi-face tracking with identity persistence: face ID assignment, score decay over time, switch cooldown to prevent jitter. Selects active speaker via scoring heuristic. More complex than SmoothedCameraman due to multi-face state management.

### 9.4 Subtitle Generation from Word Timestamps (~60 lines)

Converts faster-whisper word-level timestamps into SRT and ASS subtitle formats. Handles word grouping into display lines, timing synchronization, and ASS styling (font, color, position, background). Straightforward but fiddly timestamp arithmetic.
