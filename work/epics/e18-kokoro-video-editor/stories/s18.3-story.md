# S18.3 — /kokoro-shorts

## User Story

**As** Eduardo (content creator),
**I want** to extract video segments from approved cuts and export them as independent short files,
**so that** I have publish-ready shorts without manually editing video.

## Acceptance Criteria

```gherkin
Feature: kokoro-shorts extracts video segments from approved cuts

  Scenario: Extract segments from cuts JSON
    Given a cuts JSON file produced by kokoro-cuts
    And the original video file is accessible
    When I run /kokoro-shorts with the cuts JSON path and video path
    Then each approved cut is extracted as an independent video file
    And each file duration matches the cut boundaries (within 1s tolerance)

  Scenario: Timestamp resolution from sentence boundaries
    Given cuts JSON contains start_sentence and end_sentence (not timestamps)
    And a timestamped transcript is available (or can be generated)
    When kokoro-shorts resolves sentence boundaries to timestamps
    Then each cut maps to accurate start/end times in the video
    And resolution errors are reported clearly if a sentence cannot be located

  Scenario: Optional vertical reformat for shorts/reels
    Given an extracted segment in its original aspect ratio
    When the user requests 9:16 vertical format
    Then the segment is reformatted to 9:16 (1080x1920)
    And the main content area is preserved (center crop or letterbox)

  Scenario: Fade in/out applied
    Given an extracted video segment
    When ffmpeg processes the segment
    Then a short fade-in (0.5s) is applied at the start
    And a short fade-out (0.5s) is applied at the end

  Scenario: Output files are organized and named
    Given 3 approved cuts extracted successfully
    When the extraction completes
    Then files are saved as {basename}-short-01.mp4, {basename}-short-02.mp4, etc.
    And a summary JSON is saved with paths and metadata for each output file

  Scenario: Partial failure does not block other cuts
    Given 5 cuts to extract and 1 fails (e.g., timestamp resolution error)
    When kokoro-shorts processes all cuts
    Then the 4 successful cuts are still exported
    And the failed cut is reported with the reason
```

## Dependencies

- `kokoro-cuts` (S18.2, done) — provides cuts JSON with sentence boundaries
- `kokoro-listen` (existing) — provides transcript; may need verbose_json for word-level timestamps
- `ffmpeg` — installed on Eduardo's machine
- Test video and transcript available from S18.2 work
