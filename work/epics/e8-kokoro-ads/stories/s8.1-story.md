# Story: Skill kokoro-ads

## User Story
As Eduardo (marketing strategist),
I want a /kokoro-ads skill that receives a creative image and generates all Meta Ads deliverables,
so that I can create campaigns without re-explaining the process each session.

## Acceptance Criteria

### Scenario: Generate complete deliverables from a creative
```gherkin
Given a creative image and a client context file (contexto.md)
When I invoke /kokoro-ads
Then the skill describes the creative, identifies the audience, and generates:
  - 5 headlines in plain .txt
  - 5 primary text options in plain .txt
  - WhatsApp template in plain .txt
  - Advantage+ audience description in plain .txt
And all files are saved in the client's campaign folder
```

### Scenario: No client context available
```gherkin
Given a creative image but no contexto.md for the client
When I invoke /kokoro-ads
Then the skill asks for client name and project details before generating copy
```

### Scenario: Multiple creatives in one session
```gherkin
Given a creative already processed as creativo-01
When I provide a second creative image
Then the skill saves it as creativo-02 with auto-incrementing nomenclature
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| Konecta Park broker creative + contexto.md | /kokoro-ads | creativo-01-brokers.txt with 5 titles + 5 descriptions |
| Same session, second creative | /kokoro-ads | creativo-02-*.txt auto-numbered |
| No contexto.md exists | /kokoro-ads | Skill asks for client context first |

## Notes
- Output must be .txt plain text, never markdown — Meta Ads Manager breaks markdown formatting
- Follow Kokoro vocabulary: inversion (not precio), creacion (not producto), invitado (not cliente)
- Process: describe creative → read client context → identify audience → generate copy
- Skill file: .claude/commands/kokoro-ads.md
- Knowledge file: .claude/knowledge/kokoro-ads-meta.md
- Must mirror to extension/.claude/commands/ and extension/.claude/knowledge/
