# Story: Tests kokoro-ads

## User Story
As a developer maintaining Kokoro skills,
I want kokoro-ads registered in the test suite with structural validation,
so that regressions are caught automatically.

## Acceptance Criteria

### Scenario: kokoro-ads passes existing test patterns
```gherkin
Given kokoro-ads is added to COACHING_SKILLS, VALID_COMMANDS, EXISTING_COMMANDS
When the full test suite runs
Then all tests pass including the new skill
```

### Scenario: kokoro-ads has required structural elements
```gherkin
Given the kokoro-ads command file
When checked for output template, cross-references, and anti-vocabulary
Then it passes all structural checks
```

## Examples

| Input | Expected |
|-------|----------|
| kokoro-ads added to COACHING_SKILLS | test_has_output_template passes |
| kokoro-ads added to VALID_COMMANDS | test_cross_refs_resolve passes |
| kokoro-ads added to EXISTING_COMMANDS | file existence check passes |

## Notes
- Only modify tests/test_output_structure.py to register the new skill
- Do NOT modify the skill files themselves
