# S8.1 Design — /kokoro-ads Skill

## Problem & Value

**Problem:** Every time Eduardo creates Meta Ads campaigns, he must re-explain his entire creative-first process from scratch: describe the creative image, identify the audience from the creative (not the other way around), apply Kokoro vocabulary, and produce plain text deliverables that work inside Meta Ads Manager. This process knowledge lives in his head and in scattered feedback notes, not in a repeatable skill.

**Value:** A `/kokoro-ads` skill codifies Eduardo's 3-step process (describe creative, identify audience, generate copy) into a repeatable command that produces all Meta Ads deliverables as .txt files saved directly in the client's campaign folder. Eliminates re-explanation overhead and enforces the creative-first methodology captured in feedback_meta_ads_process.md.

## Approach

Create 4 files — a command file and a knowledge file, each mirrored to the extension directory. Unlike other Kokoro skills that guide interactive sessions, this skill is a **production workflow**: it reads an image, processes context, and generates output files in the filesystem.

The command file follows the established Kokoro skill pattern (context section, Proyector strategy, exercises/steps, output template, notes for Claude, persistencia). The knowledge file documents Meta Ads platform constraints (character limits, format requirements) and WhatsApp Business template structure.

Key difference from other skills: output is .txt plain text files saved to the client campaign folder, not markdown displayed in conversation. This is because Meta Ads Manager strips/breaks markdown formatting.

The skill references `kokoro-ads-meta.md` as its knowledge file. Eduardo's feedback (feedback_meta_ads_process.md) is the primary process source: describe the creative first, identify the audience from the creative, THEN generate copy that speaks TO that audience.

## Content Structure

### Command file (kokoro-ads.md)

| Section | Content | Key Concepts |
|---------|---------|-------------|
| Contexto | Purpose, creative-first philosophy | Describe before generating, audience from creative |
| Estrategia del Proyector | Ask permission before guiding | Proyector strategy, invitation |
| Paso 1: Describir el Creativo | Detailed description of visual elements | Colors, layout, text, imagery, mood |
| Paso 2: Leer Contexto del Invitado | Read contexto.md from campaign folder | Client context, project details, audience segments |
| Paso 2b: Sin Contexto | Fallback when no contexto.md exists | Ask for client name, project, audience |
| Paso 3: Identificar Audiencia | Determine target audience from creative + context | Who does this creative speak to? |
| Paso 4: Generar Entregables | Generate all deliverables | Headlines, primary texts, WhatsApp, Advantage+ |
| Plantilla de Salida | .txt output template for each deliverable | Plain text format, no markdown |
| Nomenclatura | Auto-increment file naming | creativo-01, creativo-02 pattern |
| Notas para Claude | Behavioral constraints | Vocabulary, anti-patterns, format rules |

### Knowledge file (kokoro-ads-meta.md)

| Section | Content |
|---------|---------|
| Meta Ads Character Limits | Headline: 40 chars, Primary text: 125 visible / 2000 max, Description: 30 chars |
| Headline Best Practices | Short, punchy, benefit-oriented, Kokoro vocabulary |
| Primary Text Best Practices | Paragraph format, line breaks for readability, hook in first line |
| WhatsApp Business Template | Greeting message + FAQ buttons structure |
| Advantage+ Audience Description | Text for Meta AI targeting, descriptive not demographic |
| Output Format Rules | Plain .txt, no markdown, no emojis unless requested |
| Kokoro Vocabulary Table | Full substitution table for Meta Ads context |
| Anti-patterns | Generic ad copy, clickbait, urgency tactics, markdown in output |

## Acceptance Criteria

**MUST:**
1. Command file follows Kokoro skill pattern (context, Proyector strategy, steps, output template, notes for Claude)
2. Knowledge file documents Meta Ads specs and constraints (character limits, format rules)
3. Output template produces .txt plain text, not markdown
4. Skill instructs 3-step process: describe creative, identify audience, generate copy
5. Extension mirror files are identical copies

**SHOULD:**
6. Auto-increment nomenclature (creativo-01, creativo-02)
7. Client context reading from contexto.md
8. Fallback flow when no contexto.md exists

**MUST NOT:**
9. Never output markdown formatting that breaks in Meta Ads Manager
10. Never skip creative description step
11. Never generate copy without reading client context first
12. Never use precio/producto/cliente/gratis/descuento — always Kokoro vocabulary

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | .txt plain text output, not markdown | Meta Ads Manager strips/breaks markdown. Eduardo's feedback explicitly requires plain text. | Markdown with copy instructions |
| D2 | 3-step sequential process (describe, audience, copy) | Captured in feedback_meta_ads_process.md. Creative description MUST happen before copy generation to ground the copy in the actual visual. | Direct copy generation from image |
| D3 | Save files to client campaign folder | Eduardo works with organized client folders. Files must be findable and versioned per creative. | Display in conversation only |
| D4 | Knowledge file separate from command | Meta Ads specs change (character limits, features). Separating knowledge allows updates without touching the flow. | Inline specs in command file |
| D5 | Mirror to extension/ directory | Established pattern — all Kokoro skills exist in both .claude/ and extension/.claude/. | Single location |
