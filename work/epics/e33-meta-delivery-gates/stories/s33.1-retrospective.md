## Reflections

### What did you learn?

Meta's delivery system thresholds are well-documented by Meta but were not
integrated into any Kokoro knowledge file. The gap was real — the motivating
bug (lunitomx/kokoro#1, 283 impressions flagged for shutdown) proved that
without this reference, premature kill decisions happen. The fix is a
standalone reference file that any Kokoro skill can consume.

### What would you change?

Nothing significant. The research agent produced actionable data that mapped
directly into the knowledge file structure. The 3-task linear chain
(structure → content → mirror) was the right decomposition for a content story.

### Framework improvements?

The research-before-implement pattern (parallel agent for thresholds) saved
time. Reinforces the knowledge-then-command pipeline: build the reference
material first, then build skills that consume it.

### What are you more capable of?

Building knowledge files from multi-source research (official Meta docs +
media buyer consensus + local knowledge from past campaigns). The triangulation
pattern produces higher-confidence thresholds than any single source.
