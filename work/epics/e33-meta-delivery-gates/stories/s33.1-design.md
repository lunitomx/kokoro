# S33.1 Design — Meta Delivery System Knowledge File

## Problem & Value

Kokoro lacks knowledge about Meta's ad delivery phases (exploration, learning,
optimization) and the thresholds that determine when an ad has sufficient data
for on/off decisions. This caused a real bug: an ad with only 283 impressions
was recommended for shutdown (lunitomx/kokoro#1) — a premature kill that would
have wasted budget already invested in the learning phase.

**Value:** Prevents premature ad kill recommendations. Gives Kokoro
evidence-based thresholds to distinguish "not enough data" from "genuinely
underperforming." Every `/kokoro-ads` analysis will reference this file before
making pause/kill recommendations.

## Approach

Single knowledge file creation following the established pattern
(`kokoro-lean-landing.md`, `kokoro-meta-ai-ecosystem.md`). The file documents
Meta's delivery system from Eduardo's perspective — not as a Meta manual, but
as strategic intelligence for the entrepreneur who needs to understand why
patience is part of the strategy.

Sources:
- Meta Business Help Center (official thresholds)
- Meta Performance Marketing documentation (delivery phases)
- Media buyer practitioner knowledge (practical thresholds)
- Existing `kokoro-meta-ai-ecosystem.md` (4 AI systems context)

The file follows the pattern of `kokoro-meta-ai-ecosystem.md` — a reference
document that skills can load for context. Section layout:

```

  - Eduardo's perspective: patience as strategic discipline

### Fase 1: Exploracion (Dias 1-3)

### Fase 2: Aprendizaje (Dias 3-14)

  - Definition, stable performance expectations
  - When to evaluate and optimize

### Hard Kill (apagar inmediatamente)

  - Condiciones especificas

  - Condiciones especificas

  - 5 dias antes de cualquier decision
  - Excepciones

  - Budget change >50%
  - Targeting change
```

## Decisions

### D1 — Eduardo's voice, not a Meta manual

**Choice:** Write the knowledge file as strategic intelligence from Eduardo's
perspective, not as a copy of Meta's help center documentation.

**Rationale:** Kokoro uses Eduardo's voice. The knowledge file must frame
delivery phases as strategic patience, not technical jargon. The entrepreneur
needs to understand *why* to wait, not just *how long* to wait. This matches
the tone of `kokoro-meta-ai-ecosystem.md` which frames AI systems as
collaborators, not black boxes.

**Alternatives considered:**
- Pure technical reference (loses Eduardo's perspective and teaching value)
- Conversational only (loses precision needed for decision matrix)
- Hybrid chosen: Eduardo's framing + precise thresholds + actionable matrix

### D2 — Practical thresholds alongside official ones

**Choice:** Document both Meta's official thresholds (50 conv/7d) and
practitioner thresholds (hard kill at 3x CPA) in separate sections.

**Rationale:** Official thresholds tell you when the algorithm has learned.
Practical thresholds tell you when to act despite incomplete learning. Both are
needed: the official ones prevent premature kills (the bug we're fixing), the
practical ones prevent indefinite spending on dead ads.

**Alternatives considered:**
- Official only (leaves gap for real-world decisions)
- Practical only (lacks Meta's own guidance as anchor)
