# S4.1 Design — Conftest Consolidation

> **Story:** S4.1 | **Size:** S | **Type:** Refactoring

## Problem

10 test files each define their own `EXTENSION_DIR` and `SKILL_PATH` module-level constants. A path change requires editing 10 files. `tests/conftest.py` already provides `extension_path`, `knowledge_path`, and `commands_path` fixtures — but only 2 test files (`test_knowledge.py`, `test_knowledge_phase2.py`) use them.

## Approach — C: Modify existing class fixtures

Each test class already has a `content` fixture that reads its skill file. The change is mechanical:

1. Remove `EXTENSION_DIR` and `SKILL_PATH` module-level constants
2. Update each class's `content(self)` fixture to accept `commands_path: Path` and build the path inline
3. Update CLI test classes that reference `SKILL_PATH` directly to accept `commands_path: Path` as a test parameter
4. Update `test_skill_file_exists` methods to accept `commands_path: Path`

This approach requires zero changes to `conftest.py`.

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Approach C — modify existing class fixtures | Minimal diff, no new fixtures in conftest |
| D2 | Remove EXTENSION_DIR and SKILL_PATH entirely | They become dead code once fixtures are used |
| D3 | CLI test methods get `commands_path` parameter | Pytest injects conftest fixtures into method signatures |
