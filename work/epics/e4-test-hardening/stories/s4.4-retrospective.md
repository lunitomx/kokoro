# S4.4 Retrospective: Output Testing Spike

> **Story:** S4.4 | **Size:** S | **Actual:** S
> **Started:** 2026-03-24 | **Completed:** 2026-03-24

## Summary

Designed and implemented a PoC for testing skill output template structure
without live Claude invocation. 40 parametrized tests across 3 categories
covering all 8 coaching skills. Found and fixed 1 content gap.

## Metrics

| Metric | Value |
|--------|-------|
| Tests added | 40 |
| Skills covered | 8/8 coaching skills |
| Content fixes | 1 (kokoro-mountain missing Siguiente paso) |
| Findings | 3 (mountain gap, canvas multi-block, validate forward-ref) |
| Total suite | 313 tests, all passing |

## What Went Well

- **RED phase revealed real issues** — the spike found a genuine content gap
  in kokoro-mountain before any user hit it
- **Parametrized design scales** — adding new skills is trivial (1 string)
- **Cross-reference validation** is valuable — catches broken links that are
  otherwise silent failures

## What Could Improve

- The "longest block" heuristic for finding output templates is fragile.
  If a skill has a longer non-output code block, it would pick wrong.
  A marker comment like `<!-- OUTPUT TEMPLATE -->` would be more reliable
  but adds ceremony to skill files.

## Patterns

- **Output template as contract** — the code block in a skill file IS the
  output specification. Testing it is testing the contract, not the
  implementation. Extension of BASE-050 (templates-as-contract).
