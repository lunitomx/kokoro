# S4.2 Scope — Anti-Vocabulary Gate

> **Story:** S4.2 | **Epic:** E4 | **Size:** S

## User Story

As a product owner protecting Eduardo's voice,
I want tests that verify prohibited words are absent from skill files
So that regressions in voice quality are caught automatically.

## In Scope

- Add `test_no_prohibited_vocabulary` to all 10 skill test files
- Prohibited words: cliente, producto, precio, gratis, descuento
- Line-level exclusion for anti-pattern context (lines with "nunca", "no uses", "no digas")
- Fix any skill files that currently contain prohibited words

## Out of Scope

- Changing assertion thresholds (that's S4.3)
- New skill files
- Modifying conftest.py

## Done When

- All 10 skill test files have anti-vocabulary test
- Zero prohibited words in skill file content (outside anti-pattern context)
- All tests pass, pyright + ruff clean
