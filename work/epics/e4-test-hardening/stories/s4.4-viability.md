# S4.4 Viability Assessment: Output Template Testing

## Verdict: VIABLE

Output template structure testing is viable and valuable. The PoC caught
a real content gap (kokoro-mountain missing Siguiente paso) and validated
structural consistency across all 8 coaching skills.

## What Works

1. **Template extraction** — fenced code blocks reliably contain output templates
2. **Structural assertions** — ## headers, tables, Siguiente paso are consistent patterns
3. **Cross-reference validation** — catches broken /kokoro-X references immediately
4. **Parametrized approach** — adding a new skill = adding 1 string to COACHING_SKILLS list

## Effort to Scale

| Action | Effort | Value |
|--------|--------|-------|
| Add Phase 3/4 skills to list | Trivial (1 line each) | High — auto-covered |
| Add more structural checks | Low (1 test method) | Medium — diminishing returns |
| Per-skill template assertions | Medium (custom per skill) | Low — brittle |

## Limitations

1. **Cannot test output quality** — only structure, not content richness
2. **Longest-block heuristic** — assumes the output template is the longest code block
3. **Forward references** — skills that reference future phases can't file-check
4. **No live output testing** — still requires Claude invocation (out of scope, expensive)

## Recommendation

Keep the 40 tests as-is. When E5/E6 add new skills, add them to
COACHING_SKILLS and move their names from VALID_COMMANDS to
EXISTING_COMMANDS. No new infrastructure needed.
