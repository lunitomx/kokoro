# Design: S14.1 — Knowledge Curation

## Design Decision: Knowledge File Format

Each knowledge file is structured for AI consumption — skills read these files
to inform their guidance. The format prioritizes:

1. **Scannable structure** — headers, tables, numbered lists
2. **Actionable frameworks** — decision trees, scoring matrices, step-by-step processes
3. **Concrete examples** — real brand cases that illustrate principles
4. **Anti-patterns** — explicit "do NOT" guidance to prevent generic responses

### Template Structure

```markdown
---
knowledge_id: "luxelling-{topic}"
purpose: "One-line purpose for skill consumption"
sources: ["notas-eduardo/XX", "workbook/XX"]
consumed_by: ["/kokoro-luxury-{skill}"]
---

# {Topic Title}

> Purpose statement

## Core Framework
{Tables, matrices, decision trees}

## Key Principles
{Numbered actionable principles}

## Case Studies
{2-3 concrete examples}

## Application Guidelines
{How skills should use this knowledge}

## Anti-patterns
{What to avoid}
```
