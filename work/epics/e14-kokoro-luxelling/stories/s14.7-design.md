# Design: S14.7 — Luxury Layers Knowledge File

## Design Decision: Additive Context Pattern

The luxury layers file follows AD-5 from the epic scope: base skills are NOT
modified. Instead, a single knowledge file provides supplementary guidance
that any skill can reference when it detects `positioning_tier = luxury` in
the client profile.

### Per-Skill Mapping

| Base Skill | Primary Source Modules | Key Shift |
|---|---|---|
| /kokoro-diagnose | luxelling-assessment | Add F-S-E positioning check before diagnosis |
| /kokoro-canvas | luxelling-scarcity, luxelling-growth | Constrain blocks with scarcity and distribution limits |
| /kokoro-finance | luxelling-pricing | Value-based pricing replaces cost-plus; Veblen applies |
| /kokoro-pescar | luxelling-communication | Seduce, don't sell; luxury visual codes |
| /kokoro-launch | luxelling-scarcity, luxelling-communication | Slow-burn launch; commercial apnea; invitation-only |
| /kokoro-factory | luxelling-scarcity, luxelling-growth | Hyper-selective distribution; curation > volume |
| /kokoro-funnel | luxelling-experience, luxelling-communication | Seduction funnel; nurturing as ritual |
| /kokoro-mafia | luxelling-quality-symbolism, luxelling-scarcity | Real exclusivity; access rituals; symbolic power |

### Structure

Each skill section contains: blockquote summary, 3-5 numbered considerations
with [source module] references, and a "Key Shift" describing the fundamental
change in approach. Closing Quick Reference Matrix for scanning.
