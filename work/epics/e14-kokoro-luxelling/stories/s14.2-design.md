# Design: S14.2 — /kokoro-luxury-assess

## Integration Points

### Reads
- `.claude/knowledge/luxelling/luxelling-assessment.md` — F-S-E framework
- `.kokoro/clients.json` — Client registry for resolution and persistence

### Writes
- `ClientProfile.metadata["positioning_tier"]` — "luxury" | "premium" | "standard"
- `ClientProfile.metadata["luxury_assessment"]` — Full assessment detail
- `ClientProfile.metadata["session_log"]` — Assessment session entry

## Flow

1. Client resolution from `.kokoro/clients.json`
2. Proyector opening — ask permission
3. 5 questions one at a time, conversationally (binary 0/1)
4. Score sum: 4-5 luxury, 2-3 premium, 0-1 standard
5. Result with F-S-E interpretation
6. Persist tier + assessment + session log
7. Next steps based on tier

## Decisions

- Conversational not form: one question at a time with Eduardo's voice
- Binary scoring: 0 or 1 per question
- No hard gate: assessment is compass, not barrier (AD-2)
