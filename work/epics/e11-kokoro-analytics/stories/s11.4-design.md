# Design: S11.4 — Traduccion de metricas

## Complexity: Simple

| Criterion | Rating |
|-----------|--------|
| Components | 2 (knowledge file + extension mirror) |
| Story points | S (~30 min, per PAT-L-014) |
| External integrations | 0 (reference data only) |
| Algorithm complexity | None (content expansion, no logic) |

Gates: No HIGH RISK. No code changes. Content-only story.

## Problem & Value

**Problem:** The current glossary has 12 basic metrics. The 4 MCP servers
return 40+ distinct metrics. When `/kokoro-analytics` encounters a metric
not in the glossary (e.g., `frequency`, `reach`, `search_impression_share`),
it has no translation template and must improvise — losing consistency.

**Value:** Every metric from every platform has a pre-written Kokoro-voice
translation with contextual thresholds. Eduardo never has to guess what a
number means or how to present it to an invitado.

## Approach

Expand the existing "Glosario de Metricas" section in
`.claude/knowledge/kokoro-analytics-metrics.md` from a flat table to a
structured reference organized by platform. Each metric gets 5 fields.

### Entry Structure

For each metric:

```markdown
### {Metric Technical Name}

- **Plataforma:** Meta Ads / Google Ads / GA4 / Search Console / Multi
- **Significado:** What this metric measures in plain Spanish
- **Plantilla Kokoro:** "Template with {placeholders} in Kokoro voice"
- **Destacar cuando:** Contextual thresholds for good/bad signals
- **Metricas relacionadas:** Cross-references to complementary metrics
```

### Platform Coverage

| Platform | Metrics to Add | Already in Glossary |
|----------|---------------|-------------------|
| Meta Ads | spend, reach, frequency, actions, cost_per_action, purchase_roas, video_views, engagement | impressions, clicks, ctr, cpc, cpm |
| Google Ads | cost, conversions, conv_rate, cost_per_conversion, search_impression_share, quality_score, avg_cpc | ctr, impressions, clicks |
| GA4 | engagementRate, eventCount, totalRevenue, transactions, itemRevenue, screenPageViewsPerSession | sessions, totalUsers, newUsers, bounceRate, avgSessionDuration, screenPageViews, conversions |

### Design Decisions

#### D1: Structured entries, not flat table

**Choice:** Replace the flat table with per-metric structured blocks.
**Rationale:** 5 fields per metric do not fit in a table. Structured blocks
are scannable and extensible.
**Trade-off:** Longer file, but knowledge files are reference data loaded
on demand — length is not a concern.

#### D2: Organize by platform, not alphabetically

**Choice:** Group metrics by platform (Meta Ads, Google Ads, GA4, GSC).
**Rationale:** When `/kokoro-analytics` processes a response from a specific
MCP server, it needs the translations for that platform's metrics quickly.
Platform grouping matches the query flow.

#### D3: Contextual thresholds, not absolute numbers

**Choice:** Thresholds use ranges and context ("above 2% is healthy for
most industries, above 5% is exceptional").
**Rationale:** A 1% CTR is terrible for search ads but acceptable for
display. Absolute thresholds would mislead. Contextual guidance lets
Eduardo apply judgment per invitado.

#### D4: Unknown metric fallback protocol

**Choice:** Add a section with instructions for handling metrics not in
the glossary — present the raw name, describe it generically, flag it for
future glossary addition.
**Rationale:** No glossary is exhaustive. GA4 custom metrics and new
platform features will produce unknown metrics. The skill needs a protocol
rather than failing silently.

## Acceptance Criteria

### MUST

1. **AC1:** Glossary covers all standard metrics from Meta Ads MCP tools
2. **AC2:** Glossary covers all standard metrics from Google Ads MCP tools
3. **AC3:** Glossary covers all standard metrics from GA4 common metrics
4. **AC4:** Glossary covers all standard metrics from Search Console MCP tools
5. **AC5:** Each entry has 5 fields (platform, meaning, template, threshold, related)
6. **AC6:** Unknown metric fallback protocol documented

### SHOULD

1. Templates use Kokoro vocabulary (inversion, invitado, creacion)
2. Thresholds are contextual, not absolute
3. Extension mirror matches source

### MUST NOT

1. Must NOT modify the skill file (kokoro-analytics.md)
2. Must NOT modify Python code
3. Must NOT remove existing glossary entries (expand, not replace)
