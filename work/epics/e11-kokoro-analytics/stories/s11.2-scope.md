# Scope: S11.2 — /kokoro-connect

## In Scope

- `/kokoro-connect` Claude Code skill (prompt file)
- Account discovery from 4 MCP servers:
  - Meta Ads: `list_ad_accounts`
  - Google Ads: `list_customers`
  - GA4: `get_account_summaries`
  - Search Console: `list_properties`
- Interactive mapping: user selects accounts per client
- Storage in `ClientProfile.metadata["platform_accounts"]`
- Client registry update via ClientStore
- Graceful handling of unreachable MCP servers
- Kokoro voice and Proyector strategy in all prompts

## Out of Scope

- Credential management or token refresh
- Modifying MCP server code (MétricaRadix)
- Automated account detection without user confirmation
- Analytics queries (that's S11.3)
- Web UI or dashboard
- Batch operations across all clients at once

## Done Criteria

- [ ] Skill file exists at `.claude/skills/kokoro-connect.md`
- [ ] Skill calls MCP list tools for all 4 platforms
- [ ] User can map discovered accounts to a client
- [ ] Mapping persists in `ClientProfile.metadata["platform_accounts"]`
- [ ] Re-running updates existing mapping (idempotent)
- [ ] Unreachable servers are reported without blocking the flow
- [ ] Tests pass (full suite)

## Technical Notes

### Platform Account Keys

```python
metadata["platform_accounts"] = {
    "meta_ads": "act_123456",        # Ad account ID
    "google_ads": "1234567890",      # Customer ID
    "ga4": "properties/123456",      # Property resource name
    "gsc": "https://example.com"     # Site URL
}
```

### MCP Tools to Call

| Platform | Tool | Returns |
|----------|------|---------|
| Meta Ads | `list_ad_accounts` | List of ad account IDs and names |
| Google Ads | `list_customers` | List of customer IDs and names |
| GA4 | `get_account_summaries` | Account + property hierarchy |
| Search Console | `list_properties` | Verified site URLs |

### Dependencies

- S11.1 complete (MCP servers registered and responding)
- Client graph from E9 (`ClientProfile`, `ClientStore`)
