# S11.3 — /kokoro-analytics

## User Story (Connextra)

**As** Eduardo (Kokoro operator),
**I want** to ask about a client's metrics in natural language
(e.g., "how is Invertikal doing on Meta Ads this month?"),
**so that** I can review campaign performance conversationally without
navigating dashboards or knowing platform-specific query syntax.

## Context

S11.1 registered the 4 MCP servers (Meta Ads, Google Ads, GA4, Search Console).
S11.2 created `/kokoro-connect` which maps platform accounts to clients via
`ClientProfile.metadata["platform_accounts"]`. Now we need the skill that
actually queries those accounts and presents results in Kokoro's voice.

The skill `/kokoro-analytics` should:

1. Resolve the client from the query (by name or active session)
2. Look up `platform_accounts` from the client's metadata
3. Call the appropriate MCP tools with the correct account IDs
4. Present metrics in Kokoro's luxurizante voice — translated from
   technical jargon to emprendedor-friendly language

## Acceptance Criteria

### Scenario: Query metrics for a connected client

```gherkin
Given a client "Invertikal" with platform_accounts mapped via /kokoro-connect
When Eduardo asks "how is Invertikal doing on Meta Ads?"
Then the skill resolves the client from the name
And retrieves the Meta Ads account ID from metadata["platform_accounts"]["meta_ads"]
And calls the appropriate MCP insight tools with that account ID
And presents the results in Kokoro's voice with business-friendly language
```

### Scenario: Query across multiple platforms

```gherkin
Given a client with accounts on Meta Ads, GA4, and Search Console
When Eduardo asks "give me an overview of [client]"
Then the skill queries all connected platforms
And presents a unified summary organized by platform
```

### Scenario: Client not connected

```gherkin
Given a client "NuevoCliente" with no platform_accounts
When Eduardo asks "how is NuevoCliente doing?"
Then the skill informs that no platforms are connected
And suggests running /kokoro-connect first
```

### Scenario: MCP server unreachable

```gherkin
Given a connected client whose Meta Ads server is down
When Eduardo queries Meta Ads metrics
Then the skill reports the server is unreachable
And continues with any other available platform data
```

### Scenario: Date range handling

```gherkin
Given a connected client
When Eduardo asks "how did [client] do last week on Google Ads?"
Then the skill interprets the date range from natural language
And passes appropriate date parameters to the MCP tools
```

## Non-Functional Requirements

- Response uses Kokoro vocabulary (inversion not precio, invitado not cliente)
- Metrics are translated to emprendedor language (e.g., CTR becomes
  "de cada 100 personas que vieron tu anuncio, X hicieron clic")
- The skill follows Proyector strategy (waits for the question, does not
  proactively dump data)
