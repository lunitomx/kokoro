# S11.4 — Traduccion de metricas

## User Story (Connextra)

**As** Eduardo (Kokoro operator),
**I want** every metric returned by the 4 MCP servers to have a Kokoro-voice
translation with business meaning and contextual thresholds,
**so that** I can present any metric to an invitado in emprendedor-friendly
language without guessing what the number means.

## Context

S11.3 created `/kokoro-analytics` with a basic 12-metric glossary in
`kokoro-analytics-metrics.md`. This story expands that glossary to cover ALL
metrics returned by Meta Ads, Google Ads, GA4, and Search Console MCP servers.

Each metric entry includes:
- Technical name
- Business meaning in Spanish
- Kokoro voice translation template
- When to highlight (good/bad thresholds)
- Related metrics to cross-reference

Also adds guidance for "unknown metrics" — what to do when a metric not in
the glossary appears.

## Acceptance Criteria

### Scenario: Complete metric coverage

```gherkin
Given the kokoro-analytics-metrics.md knowledge file
When I check the glossary section
Then every metric returned by Meta Ads MCP tools has an entry
And every metric returned by Google Ads MCP tools has an entry
And every metric returned by GA4 common metrics has an entry
And every metric returned by Search Console MCP tools has an entry
```

### Scenario: Each entry has full structure

```gherkin
Given a metric entry in the glossary
Then it has: technical name, business meaning, Kokoro translation template,
     highlight thresholds, and related metrics
```

### Scenario: Unknown metric guidance

```gherkin
Given a metric appears in MCP output that is NOT in the glossary
When /kokoro-analytics encounters it
Then it follows the "unknown metric" protocol to present it gracefully
```

## Non-Functional Requirements

- All translations use Kokoro vocabulary (inversion, invitado, creacion)
- Thresholds are contextual, not absolute (varies by industry)
- Extension mirror stays in sync with source file
- Tests pass (full suite)
