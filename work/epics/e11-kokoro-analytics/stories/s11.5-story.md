# S11.5 — Scorecard Cross-Platform

## User Story (Connextra)

**As** Eduardo (Kokoro operator),
**I want** a unified scorecard that queries ALL connected platforms for a
client and presents an executive summary with health indicators,
**so that** I can see the complete digital picture of an invitado in one
view — without querying each platform separately.

## Context

S11.3 created `/kokoro-analytics` which queries individual platforms. S11.5
builds on top: it queries ALL connected platforms at once and synthesizes a
single cross-platform scorecard with health indicators, trends, and
recommended actions.

The skill `/kokoro-scorecard` should:

1. Resolve the invitado from the query (by name or active session)
2. Query ALL connected platforms in parallel via MCP tools
3. Present a unified scorecard with:
   - Header: invitado name, period, platforms connected
   - Per-platform summary: key metrics, trend vs previous period
   - Cross-platform insights: traffic sources, conversion attribution,
     cost efficiency
   - Health indicators: green/yellow/red for each platform
   - Recommended actions based on data

## Acceptance Criteria

### Scenario: Full scorecard for multi-platform invitado

```gherkin
Given an invitado "Invertikal" with Meta Ads, GA4, and GSC connected
When Eduardo runs "/kokoro-scorecard Invertikal"
Then the skill queries all 3 platforms
And presents a unified scorecard with per-platform summaries
And shows health indicators (green/yellow/red) for each
And provides cross-platform insights
And suggests recommended actions
```

### Scenario: Single platform invitado

```gherkin
Given an invitado "NuevoProyecto" with only Meta Ads connected
When Eduardo runs "/kokoro-scorecard NuevoProyecto"
Then the skill queries Meta Ads only
And presents the scorecard with one platform section
And notes which platforms are not connected
```

### Scenario: Invitado not connected

```gherkin
Given an invitado "SinPlataformas" with no platform_accounts
When Eduardo runs "/kokoro-scorecard SinPlataformas"
Then the skill informs no platforms are connected
And suggests /kokoro-connect
```

### Scenario: Partial platform failure

```gherkin
Given an invitado with 3 platforms where one MCP server is down
When Eduardo runs "/kokoro-scorecard"
Then the skill reports which platform failed
And presents the scorecard with available data
And adjusts health indicator for unreachable platform to yellow
```

## Non-Functional Requirements

- Output uses Kokoro vocabulary (inversion, invitado, creacion)
- Follows Proyector strategy (waits for invitation)
- Degrades gracefully when platforms are unreachable
- Does not touch kokoro-analytics-metrics.md (S11.4 parallel)
