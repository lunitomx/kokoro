# Scope: S11.3 — /kokoro-analytics

## In Scope

- `/kokoro-analytics` Claude Code skill (prompt file)
- Client resolution from natural language query (by name or active session)
- Platform account lookup via `ClientProfile.metadata["platform_accounts"]`
- MCP tool dispatch for metrics queries:
  - Meta Ads: `get_account_insights_summary`, `get_campaign_performance`
  - Google Ads: `get_customer_insights_summary`, `get_campaign_performance`
  - GA4: `run_report`
  - Search Console: `get_performance_overview`, `get_search_analytics`
- Natural language date range interpretation (today, this week, last month, etc.)
- Cross-platform overview when user asks for general performance
- Graceful handling of unreachable MCP servers
- Kokoro voice and Proyector strategy in all output
- Basic metric translation (CTR, CPC, ROAS to emprendedor-friendly language)

## Out of Scope

- Deep metric translation with full glossary (that is S11.4)
- Cross-platform scorecard with unified KPIs (that is S11.5)
- Campaign modification or optimization actions (read-only)
- Credential management or token refresh
- Modifying MCP server code (MetricaRadix)
- Web UI or dashboard — purely conversational
- Historical trend analysis or anomaly detection
- Caching or storing query results

## Technical Notes

### MCP Tools Reference

| Platform | Tool | Use Case |
|----------|------|----------|
| Meta Ads | `get_account_insights_summary` | Overall account metrics |
| Meta Ads | `get_campaign_performance` | Per-campaign breakdown |
| Meta Ads | `get_campaigns` | Campaign list and status |
| Google Ads | `get_customer_insights_summary` | Overall account metrics |
| Google Ads | `get_campaign_performance` | Per-campaign breakdown |
| Google Ads | `get_campaigns` | Campaign list and status |
| GA4 | `run_report` | Custom metric queries |
| GA4 | `get_account_summaries` | Property discovery |
| Search Console | `get_performance_overview` | Overall search metrics |
| Search Console | `get_search_analytics` | Query-level data |

### Client Resolution Flow

```
User query → extract client name → ClientStore.get(name)
  → client.metadata["platform_accounts"]
  → dispatch MCP calls with account IDs
  → format response in Kokoro voice
```

### Dependencies

- S11.1 complete (MCP servers registered and responding)
- S11.2 complete (/kokoro-connect provides account mappings)
