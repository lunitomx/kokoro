# S11.1 Design — Register MCP Servers

## Problem

mcpfbads (Facebook Ads) and mcpgoogleads (Google Ads) MCP servers from
MetricaRadix are not registered in Claude Code. Their tools are unavailable
to Kokoro for analytics queries.

## Value

Once registered, all 44 MCP tools across 4 platforms become available:
- google-analytics (already registered)
- google-search-console (already registered)
- facebook-ads (this story)
- google-ads (this story)

## Approach

Register both servers using `claude mcp add` CLI commands. This is the same
mechanism used for the existing GA4 and GSC servers.

### Registration Commands

**Facebook Ads:**
```
claude mcp add facebook-ads -- uv --directory /Users/soyahuehuetedigital/Documents/MétricaRadix/mcpfbads run src/server.py
```

**Google Ads:**
```
claude mcp add google-ads -- uv --directory /Users/soyahuehuetedigital/Documents/MétricaRadix/mcpgoogleads run src/server.py
```

### Key Details

- **Command**: `uv` (manages ephemeral venv from pyproject.toml)
- **Args**: `--directory <path> run src/server.py`
- **Env vars**: None in Claude Code config — both servers use `python-dotenv`
  to load `.env` files from their own directories
- **Server names**: `facebook-ads` and `google-ads` (consistent with existing
  `google-analytics` and `google-search-console` naming)

### Components

| Component | Action | Notes |
|-----------|--------|-------|
| Claude Code MCP registry | Modify (add 2 entries) | Via `claude mcp add` |

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | `uv --directory` to run servers | Both have pyproject.toml, no venv exists, uv creates ephemeral env |
| D2 | `claude mcp add` CLI, not manual JSON | Claude Code manages config internally; no mcp.json to edit |
| D3 | Names: facebook-ads, google-ads | Consistent with existing platform-oriented naming |
| D4 | No env vars in Claude Code config | Servers load .env internally via python-dotenv |

## Acceptance Criteria

- **AC1**: mcpfbads server registered in Claude Code MCP config
- **AC2**: mcpgoogleads server registered in Claude Code MCP config
- **AC3**: Both servers' tools appear in Claude Code's available tools list

## Must Not

- No credentials stored in Claude Code MCP config
- No modifications to MetricaRadix code
