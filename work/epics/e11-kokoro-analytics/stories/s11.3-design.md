# Design: S11.3 — /kokoro-analytics

## Complexity: Moderate-Complex

| Criterion | Rating |
|-----------|--------|
| Components | 4 (skill file, knowledge file, extension mirrors, test update) |
| Story points | M (~5-8) |
| External integrations | 4 MCP servers |
| Algorithm complexity | Custom logic (query routing + date interpretation) |

Gates: No HIGH RISK. UX patterns established (kokoro-ads, kokoro-connect).
No integration gate (read-only skill, no dogfood label).

## Problem & Value

**Problem:** Eduardo has client accounts mapped via `/kokoro-connect` but no
conversational way to query their metrics. He must navigate platform dashboards
or know MCP tool syntax to get performance data.

**Value:** Eduardo asks "how is Invertikal doing on Meta Ads?" and gets an
answer in Kokoro voice — no dashboard, no syntax, no context-switching. This is
the core value delivery of E11.

## Approach

Create a Claude Code command file (`.claude/commands/kokoro-analytics.md`) that:

1. Resolves the client from the natural language query (by name or active session)
2. Reads `ClientProfile.metadata["platform_accounts"]` to discover connected platforms
3. Dispatches to the correct MCP tools based on query intent
4. Presents metrics in Kokoro voice with basic metric translation

A companion knowledge file (`.claude/knowledge/kokoro-analytics-metrics.md`)
provides the MCP tool reference, metric translation glossary, and date range
interpretation guide.

### Components

| Path | Action | Description |
|------|--------|-------------|
| `.claude/commands/kokoro-analytics.md` | create | Main skill file — query routing + presentation |
| `.claude/knowledge/kokoro-analytics-metrics.md` | create | MCP tool reference, metric glossary, date ranges |
| `extension/.claude/commands/kokoro-analytics.md` | create | Extension mirror |
| `extension/.claude/knowledge/kokoro-analytics-metrics.md` | create | Extension mirror |
| `tests/test_output_structure.py` | modify | Add kokoro-analytics to COACHING_SKILLS, VALID_COMMANDS, EXISTING_COMMANDS |

## Examples

### Example 1: Platform-specific query

**User says:**
```
/kokoro-analytics Como le va a Invertikal en Meta Ads este mes?
```

**Skill behavior:**

1. Extract client name: "Invertikal"
2. Search `.kokoro/clients.json` via `find_by_name("Invertikal")`
3. Read `metadata["platform_accounts"]["meta_ads"]` -> `"act_123456789"`
4. Extract intent: Meta Ads overview, date range = this month
5. Call: `mcp__facebook-ads__get_account_insights_summary(account_id="act_123456789", date_range="this_month")`
6. Call: `mcp__facebook-ads__get_campaign_performance(account_id="act_123456789", date_range="this_month")`
7. Present in Kokoro voice:

```
## Invertikal — Meta Ads, Marzo 2026

La presencia de Invertikal este mes muestra movimiento interesante:

**Alcance e impacto**
- 45,230 personas vieron las creaciones de Invertikal
- De cada 100 personas que vieron un anuncio, 3.2 hicieron clic (CTR 3.2%)
- 1,447 visitas generadas a traves de Meta

**Inversion y retorno**
- Se han invertido $12,450 MXN este mes
- Cada clic costo en promedio $8.60 (CPC)
- El costo por cada mil impresiones fue de $275 (CPM)

**Campanas activas**
| Campana | Estado | Inversion | Resultados |
|---------|--------|-----------|------------|
| Konecta Park — Brokers | Activa | $7,200 | 892 clics |
| Konecta Park — Inversionistas | Activa | $5,250 | 555 clics |

Desde la montana, el flujo de personas hacia Invertikal es constante.
¿Quieres profundizar en alguna campana en particular?
```

### Example 5: Date range interpretation

|-----------|---------------|
| "este mes" / "this month" | date_range: current month start to today |
| "la semana pasada" / "last week" | last 7 days (Mon-Sun) |

| "marzo" / "March" | full month of March |
| (no date mentioned) | default: last 7 days |

## Data Structures

### Query Intent Resolution

The skill must interpret the user's natural language query to determine:

```
QueryIntent:
  client_name: str          # extracted from query
  platforms: list[str]      # ["meta_ads"] or ["all"] for overview
  date_range: str           # "this_month", "last_7_days", etc.
  depth: str                # "overview" or "detailed"
  specific_campaign: str?   # if user asks about a specific campaign
```

This is NOT a Pydantic model — it's a mental model for the skill prompt. The
skill file instructs Claude to extract these dimensions from the query.

### Platform-to-MCP Tool Mapping

| Platform | Overview Tool | Detail Tool |
|----------|--------------|-------------|
| meta_ads | `get_account_insights_summary` | `get_campaign_performance` |
| google_ads | `get_customer_insights_summary` | `get_campaign_performance` |
| ga4 | `run_report` (sessions, users, sources) | `run_report` (custom dimensions) |
| gsc | `get_performance_overview` | `get_search_analytics` |

### Metric Translation (Basic — S11.4 expands this)

| Technical | Kokoro Translation |
|-----------|--------------------|
| CTR | "de cada 100 personas que vieron el anuncio, X hicieron clic" |
| CPC | "cada clic costo en promedio $X" |
| CPM | "el costo por cada mil impresiones fue $X" |
| ROAS | "por cada peso invertido, regresaron $X" |
| Impressions | "N personas vieron las creaciones" |
| Clicks | "N visitas generadas" |
| Bounce rate | "de cada 100 visitantes, X se fueron sin explorar" |

## Acceptance Criteria

### MUST

1. **AC1:** Skill file at `.claude/commands/kokoro-analytics.md` follows Kokoro patterns (Proyector strategy, luxurizante vocabulary, client resolution)
2. **AC2:** Knowledge file at `.claude/knowledge/kokoro-analytics-metrics.md` documents MCP tool mapping, metric glossary, and date range interpretation
3. **AC3:** Skill resolves client name to `platform_accounts` and dispatches correct MCP tools per platform
4. **AC4:** Cross-platform overview queries all connected platforms and presents unified summary
5. **AC5:** Missing `platform_accounts` triggers suggestion to run `/kokoro-connect`

### SHOULD

1. Unreachable MCP servers reported gracefully without blocking other platforms
2. Natural language date ranges interpreted correctly (este mes, semana pasada, etc.)
3. Extension mirrors exist and match source files

### MUST NOT

1. Must NOT modify MCP server code (MetricaRadix)
2. Must NOT modify ClientProfile model or store code
3. Must NOT create campaign modifications (read-only)
4. Must NOT dump raw MCP JSON — always translate to Kokoro voice

## Design Decisions

### D1: Skill file (prompt), not Python code

**Choice:** Claude Code command file (.md), consistent with all Kokoro skills.
**Rationale:** The "query routing" is Claude interpreting natural language — no
deterministic code needed. The skill instructs Claude which MCP tools to call
for which platforms.
**Alternatives:** Python CLI wrapper, MCP aggregation layer.

### D2: Knowledge file for MCP tool reference + metric glossary

**Choice:** Separate `.claude/knowledge/kokoro-analytics-metrics.md` file.
**Rationale:** The MCP tool mapping and metric translation table are reference
data that the skill file needs. Keeping them in a knowledge file avoids bloating
the skill prompt and allows reuse by S11.4 (metric translation expansion).
**Alternatives:** Inline in skill file, separate Python module.

### D3: Default date range = last 7 days

**Choice:** When no date is mentioned, query the last 7 days.
**Rationale:** Most useful window for a quick check. Too short (yesterday)
misses patterns. Too long (30 days) is slow and overwhelming.
**Alternatives:** Last 30 days, today only, ask the user.

### D4: Basic metric translation now, deep translation in S11.4

**Choice:** Include a basic translation glossary (CTR, CPC, CPM, ROAS) in the
knowledge file. S11.4 will expand this to a full glossary with contextual
interpretation.
**Rationale:** Delivers immediate value without blocking on the full glossary.
The skill can present metrics in Kokoro voice with just the basics.
**Alternatives:** Wait for S11.4 before implementing analytics, full glossary now.

### D5: Client resolution follows established E10 pattern

**Choice:** Same pattern as kokoro-ads and kokoro-connect — search clients.json
by name, offer /kokoro-client if not found.
**Rationale:** Proven, consistent, expected by the user.
**Alternatives:** Require client ID, use active session only.
