# Design: S11.2 — /kokoro-connect

## Summary

Create a `/kokoro-connect` Claude Code skill that guides users through
discovering platform accounts from 4 MCP servers and mapping them to
`ClientProfile` entries in the client graph. This is a prompt file (not
Python code), consistent with all other Kokoro skills.

## Components

| File | Action | Purpose |
|------|--------|---------|
| `.claude/commands/kokoro-connect.md` | CREATE | Main skill — guided onboarding flow |
| `.claude/knowledge/kokoro-connect-platforms.md` | CREATE | Platform reference (ID formats, MCP tools) |
| `extension/.claude/commands/kokoro-connect.md` | CREATE | Extension mirror |
| `extension/.claude/knowledge/kokoro-connect-platforms.md` | CREATE | Extension mirror |
| `tests/test_output_structure.py` | MODIFY | Add to COACHING_SKILLS, VALID_COMMANDS, EXISTING_COMMANDS |

## Flow

```
Step 1: Client Resolution
  - Ask which invitado to connect
  - Search .kokoro/clients.json via find_by_name
  - If not found → offer /kokoro-client
  - If found → show summary (name, group, segments)

Step 2: Platform Discovery
  - Call MCP tools on each server:
    facebook-ads    → list_ad_accounts
    google-ads      → list_customers
    google-analytics → get_account_summaries
    google-search-console → list_properties
  - If a server is unreachable → report, continue with others

Step 3: Account Selection
  - Present discovered accounts per platform
  - User selects which accounts belong to this invitado
  - Allow skipping platforms (not all clients use all platforms)

Step 4: Persist Mapping
  - Update ClientProfile.metadata["platform_accounts"]:
    {
      "meta_ads": "act_123456",
      "google_ads": "1234567890",
      "ga4": "properties/123456",
      "gsc": "https://example.com"
    }
  - Save via ClientStore (save_registry)
  - Idempotent — re-running updates, not duplicates

Step 5: Confirmation
  - Show summary table of connected accounts
  - Suggest next steps (/kokoro-ads, /kokoro-funnel)
```

## Decisions

### D1: Skill file, not Python code

All Kokoro skills are Claude Code command files (markdown prompts). This
skill follows the same pattern. No Python module needed.

### D2: Existing MCP tools for discovery

All 4 MCP servers already expose list/discovery tools. The skill calls
them and presents results conversationally. No new API endpoints.

### D3: Store in metadata["platform_accounts"]

`ClientProfile.metadata` is `dict[str, Any]`, designed for extensible
key-value data. Platform accounts fit without model changes or migration.

### D4: Kokoro voice throughout

Uses "invitado" (not "cliente"), follows Proyector strategy (wait for
invitation), applies sprezzatura. Consistent with all Kokoro skills.

### D5: Client resolution from E10 pattern

Follows the pattern established in `kokoro-ads.md`: check clients.json,
use `find_by_name`, offer `/kokoro-client` if not found.

### D6: Graceful degradation

If an MCP server is unreachable, report which failed and continue with
available ones. The user should be able to connect whatever is available.

## Risk Assessment

- **Low risk**: No Python code changes, no model changes, no API changes
- **Mirror drift**: Extension mirrors must match source — copy after creation
- **MCP availability**: Servers must be running for testing, but the skill
  handles unavailability gracefully by design
