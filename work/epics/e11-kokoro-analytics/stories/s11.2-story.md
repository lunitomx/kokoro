# S11.2 — /kokoro-connect

## User Story (Connextra)

**As** Eduardo (Kokoro operator),
**I want** a guided onboarding skill that discovers platform accounts from MCP
servers and maps them to clients in the client graph,
**so that** subsequent analytics queries know which accounts belong to which
client without manual configuration.

## Context

S11.1 registered the 4 MCP servers (Meta Ads, Google Ads, GA4, Search Console).
The servers are live and respond. Now we need a way to associate discovered
accounts with `ClientProfile` entries via `metadata["platform_accounts"]`.

The skill `/kokoro-connect` guides a non-technical user through:

1. Listing available accounts from each MCP server
2. Selecting which accounts belong to which client
3. Storing the mapping in `ClientProfile.metadata["platform_accounts"]`
4. Updating the client registry

## Acceptance Criteria

1. **Account discovery** — The skill calls list/summary tools on each MCP
   server (list_ad_accounts, list_customers, get_account_summaries,
   list_properties) and presents discovered accounts to the user.

2. **Client selection** — The user can select an existing client from the
   client graph or create a new one during the flow.

3. **Account mapping** — Selected accounts are stored in the client's
   `metadata["platform_accounts"]` dict with platform keys:
   `meta_ads`, `google_ads`, `ga4`, `gsc`.

4. **Persistence** — The mapping is persisted via the client registry
   (ClientStore) so it survives across sessions.

5. **Idempotency** — Running `/kokoro-connect` again for the same client
   updates (not duplicates) the mapping.

6. **Error handling** — If an MCP server is unreachable, the skill reports
   which servers failed and continues with the available ones.

7. **Conversational UX** — The flow uses Kokoro's voice (luxurizante,
   Proyector strategy) — no raw technical output.
