# Design: S11.5 — /kokoro-scorecard

## Integration Decisions

### ID-1: Skill file pattern

Follow the same pattern as `/kokoro-analytics` and `/kokoro-connect`:
- Prompt file at `.claude/commands/kokoro-scorecard.md`
- Extension mirror at `extension/.claude/commands/kokoro-scorecard.md`
- No Python code — pure prompt engineering skill
- Register in test_output_structure.py COACHING_SKILLS, VALID_COMMANDS,
  EXISTING_COMMANDS

### ID-2: Client resolution

Reuse the exact same resolution flow as `/kokoro-analytics`:
1. Extract invitado name from user input
2. Search `.kokoro/clients.json` via `find_by_name`
3. Read `metadata["platform_accounts"]` for connected platforms
4. If no invitado found → suggest `/kokoro-client`
5. If no platforms connected → suggest `/kokoro-connect`

### ID-3: MCP query strategy

Query ALL connected platforms using overview-level tools only:
- Meta Ads: `get_account_insights_summary` + `get_campaign_performance`
- Google Ads: `get_customer_insights_summary` + `get_campaign_performance`
- GA4: `run_report` (sessions, users, bounce rate, channels)
- Search Console: `get_performance_overview`

Request two periods: current (last 7 days default) and previous
(7 days before that) for trend comparison.

### ID-4: Health indicator logic

Defined in the prompt as heuristic rules:
- **Green**: metrics stable or improving, campaigns active
- **Yellow**: metrics declining >10%, server unreachable, no recent data
- **Red**: metrics declining >25%, all campaigns paused, no data for >7 days

These are prompt-level guidelines, not code. Claude interprets the data
and assigns indicators based on patterns.

### ID-5: Cross-platform insights

Synthesized from all platform data:
- Traffic source distribution (paid vs organic vs direct)
- Cost efficiency comparison (Meta Ads CPC vs Google Ads CPC)
- Conversion attribution (which platform drives most conversions)
- Overall trend direction

### ID-6: Session log persistence

Same pattern as `/kokoro-analytics` — log to
`ClientProfile.metadata["session_log"]` with type "scorecard".

## Risks

1. Too many MCP calls slowing response — mitigate with overview-level tools
2. Platform data inconsistency — present each platform honestly, note gaps
3. S11.4 parallel modification of kokoro-analytics-metrics.md — we do NOT
   reference that file, avoiding collision entirely

## Structure

```
.claude/commands/kokoro-scorecard.md       # Main skill file
extension/.claude/commands/kokoro-scorecard.md  # Extension mirror
tests/test_output_structure.py             # Updated registration
```
