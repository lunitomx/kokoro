# S1.6 — /kokoro-prune

## User Story

**As** an entrepreneur using Kokoro,
**I want** a guided session to evaluate and prune my product tree,
**So that** I can make conscious decisions about which business lines to grow, maintain, transform, or prune based on revenue, profitability, and purpose alignment.

## Acceptance Criteria

### AC1: Skill File Exists
Given the kokoro extension is installed
When I check .claude/commands/
Then kokoro-prune.md exists as a valid skill file

### AC2: Product Tree Inventory
Given I run /kokoro-prune
When the session begins
Then Claude guides me through inventorying all business lines, products, and services
And uses the tree metaphor (roots, trunk, branches, fruits)

### AC3: Evaluation Matrix
Given the product inventory is complete
When the session continues
Then Claude guides me through evaluating each branch on 4 criteria (ingresos, rentabilidad, coherencia, energia)

### AC4: Classification
Given the evaluation is complete
When the session continues
Then Claude guides me through classifying each branch (crecer, mantener, transformar, podar)

### AC5: Pruning Summary
Given the classification is complete
When the session ends
Then Claude produces a structured pruning summary with decisions and transition dates

### AC6: CLI Integration
Given the kokoro package is installed
When I run kokoro init
Then the prune skill file is copied to .claude/commands/

### AC7: Tests
Given the skill implementation
When tests run
Then all skill-related tests pass
