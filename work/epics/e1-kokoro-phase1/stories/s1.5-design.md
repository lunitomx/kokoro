# S1.5 Design — /kokoro-mountain Skill

## Problem & Value

**Problem:** After diagnosing their current state with /kokoro-diagnose (S1.4), entrepreneurs need to translate awareness into a structured 3-year vision with measurable objectives. Without guidance, vision exercises stay abstract and OKRs lack specificity.

**Value:** Second Phase 1 skill. Completes the diagnostic-to-vision flow. Entrepreneurs get Eduardo's Montaña del Mañana exercise — a concrete methodology for building layered vision (summit, base camps, base) — plus OKR definition with measurable key results.

## Approach

Create `extension/.claude/commands/kokoro-mountain.md` — a markdown prompt file that Claude Code executes when a user types `/kokoro-mountain`. The prompt guides the user through:

1. **Montaña del Mañana** — build the mountain from summit to base:
   - Summit (Cima): 3-year vision — where do you want to be?
   - Base Camp 2 (Campamento 2): 1-year milestone
   - Base Camp 1 (Campamento 1): 6-month milestone
   - Base (Base): current state — where are you now?
   - Path (Sendero): the route connecting base to summit

2. **OKR Definition** — translate the mountain into measurable objectives:
   - 2-3 Objectives aligned with Base Camp 1 (nearest milestone)
   - 2-3 Key Results per Objective (measurable, time-bound)
   - Examples from Eduardo's methodology

3. **Vision Summary** — structured output combining mountain map and OKRs

The skill file references `kokoro-phase1-vision.md` as knowledge context. No CLI changes needed — `_copy_dir_no_overwrite` already handles `commands/`.

## Acceptance Criteria

**MUST:**
1. `kokoro-mountain.md` exists in `extension/.claude/commands/`
2. Skill guides Montaña del Mañana exercise (summit, base camps, base, path)
3. Skill guides OKR definition with measurable key results
4. Skill produces a structured vision summary
5. `kokoro init` copies the skill file to target `.claude/commands/`
6. All tests pass, pyright + ruff clean

**SHOULD:**
1. Skill references knowledge file for methodology depth
2. Eduardo's voice patterns are present (Projector strategy)

**MUST NOT:**
1. Hardcode specific business objectives (guided process, not prescriptions)
2. Skip the Projector strategy (ask before guiding — let the entrepreneur lead)

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Single markdown file with structured sections | PAT-L-001 skill file pattern validated in S1.4 | Multi-file skill |
| D2 | No CLI changes needed | Existing `_copy_dir_no_overwrite` already handles `commands/` | Separate registration system |
| D3 | Knowledge reference via Claude Code context loading | Claude Code loads knowledge files automatically from `.claude/knowledge/` | Inline all methodology content in skill file (too long) |
