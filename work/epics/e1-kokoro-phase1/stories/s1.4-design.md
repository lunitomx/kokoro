# S1.4 Design — /kokoro-diagnose Skill

## Problem & Value

**Problem:** Kokoro has knowledge files (S1.3) and Eduardo's brain in CLAUDE.md (S1.2), but no guided skills yet. Entrepreneurs need structured diagnostic sessions — not just a chatbot that "sounds like Eduardo," but a process that walks them through proven exercises.

**Value:** First usable skill. Proves the entire skill pattern works (markdown file -> slash command -> guided session). Entrepreneurs get a guided Speed Boat + Vision 20/20 diagnostic — the opening move of Phase 1 "Preparar el Suelo."

## Approach

Create `extension/.claude/commands/kokoro-diagnose.md` — a markdown prompt file that Claude Code executes when a user types `/kokoro-diagnose`. The prompt guides the user through:

1. **Speed Boat exercise** — identify anchors (obstacles), wind (advantages), rocks (risks)
2. **Vision 20/20 exercise** — assess clear vision (what you see well), blurry vision (uncertain areas), blind spots (what you're missing)
3. **Diagnostic summary** — structured output combining findings from both exercises into an action plan

The skill file references `kokoro-phase1-diagnostico.md` as knowledge context. Claude Code loads knowledge files automatically when present in `.claude/knowledge/`.

No CLI changes needed — the existing `_copy_dir_no_overwrite` function already copies everything under `commands/` during `kokoro init`.

## Components

## Acceptance Criteria

**MUST:**
1. `kokoro-diagnose.md` exists in `extension/.claude/commands/`
2. Skill guides through Speed Boat exercise (anchors, wind, rocks)
3. Skill guides through Vision 20/20 exercise (clear, blurry, blind spots)
4. Skill produces a structured diagnostic summary
5. `kokoro init` copies the skill file to target `.claude/commands/`
6. All tests pass, pyright + ruff clean

**SHOULD:**
1. Skill references knowledge file for methodology depth
2. Eduardo's voice patterns are present in the prompt

**MUST NOT:**
1. Hardcode specific business advice (it's a guided process, not answers)
2. Skip the Projector strategy (ask before guiding — let the entrepreneur lead)

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Single markdown file with structured sections | Matches Claude Code convention — filename = slash command | YAML config + template; multi-file skill |
| D2 | No CLI changes needed | Existing `_copy_dir_no_overwrite` already handles `commands/` | Separate registration system |
| D3 | Knowledge reference via Claude Code context loading | Claude Code loads knowledge files automatically when in `.claude/knowledge/` | Inline all methodology content in skill file (too long) |
