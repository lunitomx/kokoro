# S1.3 Design: Knowledge Files

## Complexity: Simple (2 components, 0 integrations)

## What & Why

Knowledge files provide methodology reference content that Claude Code loads as context when running Kokoro skills. Without them, skills have no framework details to guide sessions. Completes M1 Walking Skeleton.

## Approach

Markdown files with `kokoro-` prefix in `extension/.claude/knowledge/`. The prefix convention distinguishes Kokoro-owned files (overwritten on re-run) from user-created files (preserved). Modify `_copy_dir_no_overwrite` to handle this.

### Components

| Component | Change | Detail |
|-----------|--------|--------|
| `extension/.claude/knowledge/kokoro-*.md` | create | 5 methodology reference files |
| `src/kokoro/cli.py` | modify | `_copy_dir_no_overwrite` overwrites `kokoro-*` files |
| `tests/test_cli.py` | modify | Add knowledge file tests |

## Examples

### Knowledge file structure

```
extension/.claude/knowledge/
├── kokoro-metodologia.md           # 4-phase overview + principles
├── kokoro-phase1-diagnostico.md    # Speed Boat + Vision 20/20
├── kokoro-phase1-vision.md         # Montaña del Mañana + OKRs
├── kokoro-phase1-poda.md           # Prune the Product Tree
├── kokoro-phase1-finanzas.md       # Financial assessment
```

### CLI update logic

```python
# In _copy_dir_no_overwrite:
elif not target.exists() or item.name.startswith("kokoro-"):
    shutil.copy2(item, target)
```

### Knowledge file format

```markdown
<!-- kokoro-managed: do not edit, will be overwritten by kokoro init -->
# [Framework Name]

## Cuándo usar
[When to use this framework]

## Proceso
[Step-by-step process]

## Preguntas clave
[Key questions to ask during the session]
```

## Acceptance Criteria

### MUST
1. `extension/.claude/knowledge/` contains 5 methodology files
2. Each file has substantive framework content (not stubs)
3. `kokoro init` copies knowledge files on fresh install
4. `kokoro init` re-run overwrites `kokoro-*` files, preserves user files
5. All tests GREEN (existing 33 + new knowledge tests)

### MUST NOT
- Knowledge files must not duplicate CLAUDE.md content (voice/identity stays in CLAUDE.md)
- Must not break existing tests
