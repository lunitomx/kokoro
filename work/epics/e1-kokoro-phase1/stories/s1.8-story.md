# S1.8 — Meta Skills (/kokoro + /kokoro-session)

## User Story

**As** an entrepreneur using Kokoro,
**I want** a router skill that guides me to the right Phase 1 session and a session management skill that tracks my progress,
**So that** I don't need to know which specific skill to run — Kokoro meets me where I am and remembers where I've been.

## Acceptance Criteria

### AC1: Router Skill File Exists
Given the kokoro extension is installed
When I check .claude/commands/
Then kokoro.md exists as a valid skill file

### AC2: Session Skill File Exists
Given the kokoro extension is installed
When I check .claude/commands/
Then kokoro-session.md exists as a valid skill file

### AC3: Router Diagnosis
Given I run /kokoro
When the session begins
Then Claude asks diagnostic questions to understand where I am in Phase 1
And recommends the appropriate next skill (diagnose, mountain, prune, or finance)

### AC4: Router Awareness of All Skills
Given the /kokoro router
When it recommends a skill
Then it references the correct Phase 1 methodology order (Diagnóstico → Visión → Poda → Finanzas)
And it knows all 4 skill names and their purposes

### AC5: Session Start
Given I run /kokoro-session
When I'm starting a new Phase 1 journey
Then Claude initializes a session context and explains the Phase 1 roadmap
And guides me to begin with /kokoro-diagnose

### AC6: Session Continue
Given I run /kokoro-session
When I have completed one or more Phase 1 skills
Then Claude helps me review what was accomplished
And identifies the next step in the methodology

### AC7: Session Review
Given I run /kokoro-session
When I ask for a progress review
Then Claude summarizes completed phases, key findings, and remaining steps
And presents a structured Phase 1 progress report

### AC8: CLI Integration
Given the kokoro package is installed
When I run kokoro init
Then both kokoro.md and kokoro-session.md are copied to .claude/commands/

### AC9: Tests
Given the skill implementations
When tests run
Then all skill-related tests pass (content validation + CLI integration)
