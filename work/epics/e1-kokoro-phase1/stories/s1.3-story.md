# S1.3 — Knowledge Files

## User Story

**As** a Kokoro user,
**I want** `kokoro init` to install methodology reference files into `.claude/knowledge/`,
**So that** Claude Code can access Eduardo's frameworks and methodology as context when running skills.

## Acceptance Criteria

```gherkin
Scenario: Fresh install copies knowledge files
  Given a project with no .claude/ directory
  When I run `kokoro init`
  Then .claude/knowledge/ contains methodology reference files
  And files are non-empty markdown with framework content

Scenario: Existing knowledge directory preserves user files
  Given a project with .claude/knowledge/my-notes.md
  When I run `kokoro init`
  Then .claude/knowledge/my-notes.md is preserved unchanged
  And Kokoro knowledge files are added alongside

Scenario: Re-run updates knowledge files
  Given a project where `kokoro init` was already run
  When I run `kokoro init` again
  Then Kokoro knowledge files are updated to latest version
  And user-created files are preserved
```
