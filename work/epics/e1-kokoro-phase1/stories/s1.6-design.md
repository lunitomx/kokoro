# S1.6 Design — /kokoro-prune Skill

## Problem & Value

**Problem:** After diagnosing (S1.4) and building their mountain vision (S1.5), entrepreneurs need to evaluate their current product/service portfolio. Many carry unprofitable or misaligned business lines out of habit or fear. Without a structured pruning process, resources stay scattered across too many branches.

**Value:** Third Phase 1 skill. Completes the diagnostic-vision-pruning flow. Entrepreneurs get Eduardo's Arbol de Productos exercise — a concrete methodology for inventorying, evaluating, classifying, and pruning business lines based on revenue, profitability, purpose coherence, and energy.

## Approach

Create `extension/.claude/commands/kokoro-prune.md` — a markdown prompt file that Claude Code executes when a user types `/kokoro-prune`. The prompt guides the user through:

1. **Product Tree Inventory** — list all business lines/products/services using the tree metaphor (roots, trunk, branches, fruits, dry leaves)
2. **Evaluation Matrix** — evaluate each branch on 4 criteria:
   - Ingresos (revenue)
   - Rentabilidad (profitability)
   - Coherencia (alignment with purpose)
   - Energia (energy consumed vs generated)
3. **Classification** — classify each branch into one of 4 categories:
   - Crecer (grow) — high profitability + high coherence
   - Mantener (maintain) — profitable but no clear growth
   - Transformar (transform) — coherent but not profitable
   - Podar (prune) — low profitability + low coherence
4. **Pruning Summary** — structured output with decisions, dates, resource reallocation

The skill file references `kokoro-phase1-poda.md` as knowledge context. No CLI changes needed — `_copy_dir_no_overwrite` already handles `commands/`.

## Acceptance Criteria

**MUST:**
1. `kokoro-prune.md` exists in `extension/.claude/commands/`
2. Skill guides through product inventory and evaluation matrix (ingresos, rentabilidad, coherencia, energia)
3. Skill guides through classification (crecer, mantener, transformar, podar)
4. Skill produces structured pruning summary with decisions and dates
5. `kokoro init` copies the skill file to target `.claude/commands/`
6. All tests pass, pyright + ruff clean

**SHOULD:**
1. Skill references knowledge file for methodology depth
2. Eduardo's voice patterns are present (Projector strategy)

**MUST NOT:**
1. Hardcode specific pruning recommendations (guided process, not prescriptions)
2. Skip the Projector strategy (ask before guiding — let the entrepreneur lead)

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Single markdown file with structured sections | PAT-L-001 skill file pattern validated in S1.4/S1.5 | Multi-file skill |
| D2 | No CLI changes needed | Existing `_copy_dir_no_overwrite` already handles `commands/` | Separate registration system |
| D3 | Knowledge reference via Claude Code context loading | Claude Code loads knowledge files automatically from `.claude/knowledge/` | Inline all methodology content in skill file (too long) |
