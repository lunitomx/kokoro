# S1.7 Design — /kokoro-finance Skill

## Problem & Value

**Problem:** After diagnosing (S1.4), building a mountain vision (S1.5), and pruning the product tree (S1.6), entrepreneurs need financial clarity for the lines they chose to grow. Many run marketing as a gamble — spending without measuring CPA, ROI, or conversion rates. Without real numbers, strategic decisions are based on gut feeling, not data.

**Value:** Fourth and final Phase 1 skill. Completes the "Preparar el Suelo" flow: diagnose, envision, prune, evaluate financially. Entrepreneurs get Eduardo's financial assessment methodology — a structured process for inventorying financials per product, analyzing acquisition costs, understanding key metrics, and creating a 90-day investment validation plan. Marketing becomes investment, not expense.

## Approach

Create `extension/.claude/commands/kokoro-finance.md` — a markdown prompt file that Claude Code executes when a user types `/kokoro-finance`. The prompt guides the user through:

1. **Financial Inventory** — per product/service: revenue, direct costs, indirect costs, real margin, active clients
2. **Acquisition Analysis** — CPA per channel, conversion rates, cost per lead vs cost per client
3. **Key Metrics** — CPA, ROI, conversion rate, LTV with healthy benchmarks
4. **Budget Benchmarks** — sector-specific maximum investment percentages
5. **90-Day Validation Plan** — investment amount, expected return, weekly metrics, scale/pause decision criteria

The skill file references `kokoro-phase1-finanzas.md` as knowledge context. No CLI changes needed — `_copy_dir_no_overwrite` already handles `commands/`.

## Acceptance Criteria

**MUST:**
1. `kokoro-finance.md` exists in `extension/.claude/commands/`
2. Skill guides through financial inventory per product/service
3. Skill covers acquisition analysis (CPA, channels, conversion)
4. Skill references key metrics (CPA, ROI, conversion rate, LTV)
5. Skill includes sector-specific budget benchmarks
6. Skill produces 90-day validation plan
7. `kokoro init` copies the skill file to target `.claude/commands/`
8. All tests pass, pyright + ruff clean

**SHOULD:**
1. Skill references knowledge file for methodology depth
2. Eduardo's voice patterns are present (Projector strategy, vocabulary)
3. "Marketing es inversion, no gasto" principle is explicit

**MUST NOT:**
1. Hardcode specific financial recommendations (guided process, not prescriptions)
2. Skip the Projector strategy (ask before guiding)
3. Use generic marketing language ("hacks", "monetizar", "escalar rapido")

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Single markdown file with structured sections | PAT-L-001 skill file pattern validated in S1.4/S1.5/S1.6 | Multi-file skill |
| D2 | No CLI changes needed | Existing `_copy_dir_no_overwrite` already handles `commands/` | Separate registration system |
| D3 | Knowledge reference via Claude Code context loading | Claude Code loads knowledge files automatically from `.claude/knowledge/` | Inline all methodology content in skill file (too long) |
| D4 | 4 exercises matching knowledge file structure | Mirrors `kokoro-phase1-finanzas.md` sections directly | Custom exercise structure |
