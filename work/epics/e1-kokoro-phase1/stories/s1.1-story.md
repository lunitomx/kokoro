# Story: Package Skeleton

## User Story
As a developer installing Kokoro,
I want to run `pip install` and `kokoro init`,
so that the extension files are copied into my project's .claude/ directory.

## Acceptance Criteria

### Scenario: Fresh install into empty project
```gherkin
Given a project directory with no .claude/ folder
When I run `kokoro init`
Then a .claude/ directory is created with CLAUDE.md, commands/, and knowledge/
And the CLI prints a success message
```

### Scenario: Install into project with existing .claude/
```gherkin
Given a project directory with an existing .claude/CLAUDE.md
When I run `kokoro init`
Then Kokoro content is appended to CLAUDE.md between <!-- KOKORO START --> and <!-- KOKORO END --> markers
And commands/ and knowledge/ are copied without overwriting existing files
```

### Scenario: pip install from GitHub
```gherkin
Given a Python environment
When I run `pip install git+https://github.com/lunitomx/kokoro.git`
Then the kokoro package is installed
And the `kokoro` CLI command is available
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| Empty dir | `kokoro init` | .claude/ created with all extension files |
| Dir with .claude/CLAUDE.md | `kokoro init` | Kokoro appended with markers |
| `kokoro init --target /tmp/test` | Custom target | Extension installed to /tmp/test/.claude/ |

## Notes
- src layout: src/kokoro/ for proper packaging
- Extension files shipped as package_data in extension/ directory
- CLI entry point via pyproject.toml [project.scripts]
- Minimal dependencies: only stdlib (pathlib, shutil)
