# S1.8 Design — Meta Skills (/kokoro + /kokoro-session)

## Problem & Value

**Problem:** Entrepreneurs must know which `/kokoro-*` skill to run and in what order. There is no single entry point, no guidance on where to start, and no way to track progress across the Phase 1 journey. A new user faces 4 skills with no map.

**Value:** `/kokoro` becomes the single entry point — no knowledge of methodology order needed. `/kokoro-session` provides journey management: start, continue, review. Together they complete M3 (Feature Complete) and make Phase 1 a guided experience rather than a menu of disconnected tools.

## Approach

Two new skill files following the established PAT-L-001 pattern, plus a small CLI fix.

### 1. `kokoro.md` — Router Skill

A diagnostic router that asks 3-4 simple questions to determine where the entrepreneur is in Phase 1, then recommends the appropriate next skill.

**Flow:**
1. Projector strategy — ask permission to guide
2. Ask diagnostic questions (have you done X? do you have Y?)
3. Map answers to Phase 1 position
4. Recommend next skill with context on why

**Questions map to skills:**
- No diagnostic done? -> `/kokoro-diagnose`
- Diagnostic done but no vision? -> `/kokoro-mountain`
- Vision exists but too many product lines? -> `/kokoro-prune`
- Products pruned but no financial clarity? -> `/kokoro-finance`
- All 4 done? -> Celebrate + preview Phase 2

### 2. `kokoro-session.md` — Session Manager Skill

Three modes based on user context:

**Start mode:** User is beginning Phase 1 for the first time.
- Explain the Phase 1 roadmap (4 skills, their purpose, their order)
- Set expectations (time commitment, what they will produce)
- Guide to begin with `/kokoro-diagnose`

**Continue mode:** User has completed one or more skills.
- Ask what they have completed
- Review key findings from completed phases
- Identify the next step and explain its purpose
- Provide the specific command to run

**Review mode:** User wants a progress summary.
- Walk through all 4 phases: completed/pending
- For completed phases: summarize key outputs
- For pending phases: explain what they will produce
- Present a structured Phase 1 progress report

### 3. CLI Fix — `_copy_dir_no_overwrite`

The current prefix check `item.name.startswith("kokoro-")` at line 79 of `cli.py` excludes `kokoro.md` from being overwritten on subsequent runs. Change to `item.name.startswith("kokoro")` so both `kokoro.md` and `kokoro-session.md` are treated as Kokoro-owned files.

## Examples

### Router usage

```
User: /kokoro

Kokoro: "Bienvenido. Antes de guiarte, necesito entender dónde estás parado.
¿Me permites hacerte algunas preguntas?"

User: Sí, adelante.

Kokoro: "¿Ya has hecho un diagnóstico de tu negocio — identificado las anclas
que te frenan y los puntos ciegos que no ves?"

User: No, apenas estoy empezando.

Kokoro: "Perfecto. El primer paso es Preparar el Suelo, y eso empieza con
entender dónde estás parado. Te recomiendo iniciar con /kokoro-diagnose —
una sesión de diagnóstico usando Speed Boat y Visión 20/20. Ahí vamos a
hacer visible lo invisible..."
```

### Session start

```
User: /kokoro-session

Kokoro: "Bienvenido a la Fase 1: Preparar el Suelo. Antes de sembrar,
la tierra necesita estar lista.

El camino tiene 4 estaciones:
1. Diagnóstico — Speed Boat + Visión 20/20 (hacer visible lo invisible)
2. Montaña del Mañana — tu visión a 3 años + OKRs
3. Poda del Árbol — elegir qué líneas crecen y cuáles se podan
4. Evaluación Financiera — de la intuición a los números reales

¿Estás listo para comenzar? Usa /kokoro-diagnose para la primera estación."
```

### Session continue (after completing diagnose + mountain)

```
User: /kokoro-session

Kokoro: "¿En qué punto del camino estás? ¿Qué sesiones has completado?"

User: Ya hice el diagnóstico y la montaña del mañana.

Kokoro: "Excelente. Ya tienes visibilidad de tus anclas y puntos ciegos,
y una montaña clara a 3 años con OKRs. El siguiente paso es la Poda —
mirar tu árbol de productos y decidir qué ramas alimentar y cuáles cortar.

Usa /kokoro-prune para esa sesión."
```

## Acceptance Criteria

**MUST:**
1. `kokoro.md` exists in `extension/.claude/commands/`
2. `kokoro-session.md` exists in `extension/.claude/commands/`
3. Router asks diagnostic questions to identify Phase 1 position
4. Router references all 4 Phase 1 skills by name and purpose
5. Router follows methodology order: Diagnóstico -> Visión -> Poda -> Finanzas
6. Session skill supports start/continue/review flows
7. Both skills use Eduardo's voice (Projector strategy, vocabulary, sprezzatura)
8. `kokoro init` copies both skills to `.claude/commands/`
9. `kokoro init` overwrites both skills on subsequent runs (Kokoro-owned)
10. All tests pass, pyright + ruff clean

**SHOULD:**
1. Router provides context on WHY the recommended skill is next (not just which)
2. Session review produces a structured Phase 1 progress report template
3. Both skills reference knowledge files for methodology depth
4. "Siguiente paso" section in each skill points to the next logical action

**MUST NOT:**
1. Persist state to files (conversation context suffices)
2. Skip the Projector strategy (ask before guiding)
3. Use generic marketing language ("hacks", "monetizar", "escalar rápido")
4. Hardcode user progress (always ask the user where they are)
5. Reference Phases 2-4 skills by command name (those don't exist yet)

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|--------------|
| D1 | Modify `_copy_dir_no_overwrite` prefix check from `"kokoro-"` to `"kokoro"` | `kokoro.md` is Kokoro-owned and should update on reinstall. Current code skips it. | Accept asymmetric behavior; Rename to `kokoro-router.md` |
| D2 | Simple question-based routing (3-4 questions) | User self-reports progress. No persistent state needed. Matches Projector strategy (ask before guiding). | Checklist approach; File-based progress tracking |
| D3 | Session state lives in conversation context only | Scope explicitly excludes cross-session persistence. Claude Code sessions maintain context naturally. | JSON state file; Markdown progress log |
